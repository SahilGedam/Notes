1. REACT BASICS & VIRTUAL DOM ARCHITECTURE [Importance: 7.5/10]
--------------------------------------------------------------------------------
* What is React?
  React is an open-source, component-based front-end JavaScript library developed by 
  Meta (Facebook). It is designed strictly for building user interfaces (often referred 
  to as the View layer in MVC).

* Virtual DOM (VDOM) & Reconciliation:
  - Virtual DOM: A lightweight, in-memory representation of the real DOM.
  - Reconciliation (Diffing Algorithm):
    1. When a component's state or props change, React creates a new VDOM tree.
    2. React compares (diffs) this new tree with the previous VDOM tree.
    3. It calculates the minimum set of changes required.
    4. React batch-applies these changes to the real DOM (via ReactDOM), minimizing 
       expensive browser reflows and repaints.
  - Fiber Engine: The reconciliation engine introduced in React 16. It enables incremental 
    rendering—the ability to split rendering work into chunks and spread it over multiple 
    frames (concurrency).

* Single-Page Applications (SPA):
  SPAs load a single HTML document and dynamically rewrite the DOM as the user interacts 
  with the app. This avoids reloading entire web pages, yielding desktop-like performance.


2. COMPONENTS & JSX SYNTAX [Importance: 7/10]
--------------------------------------------------------------------------------
* JSX (JavaScript XML):
  JSX is a syntax extension for JavaScript that looks like HTML. It allows developers 
  to write UI structures directly alongside JavaScript logic.
  - Browser Parsing: Browsers cannot read JSX natively. Transpilers (like Babel) compile 
    JSX tags into standard JavaScript `React.createElement()` calls.
    ```javascript
    // JSX:
    const element = <h1 className="title">Hello</h1>;

    // Compiled JavaScript:
    const element = React.createElement('h1', { className: 'title' }, 'Hello');
    ```

* Functional vs. Class Components:
  - Functional Components (Modern standard): Plain JavaScript functions that accept `props` 
    and return JSX. They use **Hooks** to manage state and lifecycle.
  - Class Components (Legacy): ES6 classes extending `React.Component`. Use `this.state` 
    and lifecycle methods (`componentDidMount`, etc.).

* Fragments (`<React.Fragment>` or `<>`):
  Used to group multiple elements together without adding redundant wrapper nodes to the DOM.
  ```jsx
  return (
    <>
      <h1>First Element</h1>
      <p>Second Element</p>
    </>
  );
  ```


3. LIFECYCLE HOOKS & SIDE EFFECTS (useEffect) [Importance: 8/10]
--------------------------------------------------------------------------------
* Side Effects:
  Operations that interact with the outside world (e.g. data fetching, DOM manipulation, 
  subscriptions) must be isolated inside lifecycle methods or hooks.

* `useEffect` (The Lifecycle Workhorse):
  Combines the functionality of class lifecycle methods `componentDidMount`, 
  `componentDidUpdate`, and `componentWillUnmount` into a single hook.
  
  - Dependency Array Behaviors:
    1. No Dependency Array: Runs on *every* render.
       ```javascript
       useEffect(() => { console.log('Runs every render'); });
       ```
    2. Empty Dependency Array (`[]`): Runs exactly *once* after initial mount.
       ```javascript
       useEffect(() => { console.log('Mounted'); }, []);
       ```
    3. With Dependencies (`[count]`): Runs on mount and whenever variables in the array change.
       ```javascript
       useEffect(() => { console.log('Count updated:', count); }, [count]);
       ```

* Cleanups (Preventing Memory Leaks):
  If the hook returns a function, React runs this cleanup function when the component 
  unmounts or before re-running the effect.
  ```javascript
  useEffect(() => {
    const handleResize = () => console.log(window.innerWidth);
    window.addEventListener('resize', handleResize);

    return () => {
      // Cleanup listener
      window.removeEventListener('resize', handleResize);
    };
  }, []);
  ```

* `useLayoutEffect`:
  Identical signature to `useEffect`, but fires synchronously *after* all DOM mutations 
  but *before* the browser paints the screen. Use this only to measure DOM layout dimensions 
  to prevent visual flickers.


4. STATE MANAGEMENT & HOOKS [Importance: 8.5/10]
--------------------------------------------------------------------------------
* `useState`:
  Declares a state variable inside functional components.
  - Functional Updates: Always pass a function to updater functions when updating 
    state based on its previous value to avoid stale closure bugs.
    ```javascript
    const [count, setCount] = useState(0);
    // Correct way to increment:
    setCount(prevCount => prevCount + 1);
    ```

* `useReducer`:
  Alternative state hook suited for complex state objects and transition logic (similar to Redux).
  ```javascript
  const reducer = (state, action) => {
    switch (action.type) {
      case 'increment': return { count: state.count + 1 };
      default: return state;
    }
  };
  const [state, dispatch] = useReducer(reducer, { count: 0 });
  ```

* Immutability in React State:
  Never mutate state directly. Always return new references to arrays or objects.
  ```javascript
  // ❌ BAD: Mutates existing array
  list.push(newItem);
  setList(list);

  // ✅ GOOD: Spreads and creates new array reference
  setList([...list, newItem]);
  ```


5. COMPONENT COMMUNICATION [Importance: 8/10]
--------------------------------------------------------------------------------
* Data Flow:
  Data always flows unidirectionally from parent to child (via `props`).

* Parent to Child (Props):
  ```jsx
  function Parent() {
    return <Child message="Hello from Parent" />;
  }
  ```

* Child to Parent (Callback functions):
  Child triggers a callback function provided by the parent, passing data back up.
  ```jsx
  function Parent() {
    const handleData = (childData) => console.log(childData);
    return <Child onSendData={handleData} />;
  }
  ```

* Lifting State Up:
  When two sibling components need to share data, hoist the state up to their nearest 
  common ancestor component, then pass it down via props.

* Children Prop (`props.children`):
  Allows components to project arbitrary JSX content, similar to Angular's `<ng-content>`.
  ```jsx
  function Wrapper({ children }) {
    return <div className="border">{children}</div>;
  }
  ```


6. CONTEXT API & GLOBAL STATE [Importance: 8/10]
--------------------------------------------------------------------------------
* Context API:
  Enables sharing state globally across components without manual "prop-drilling" through 
  intermediate levels.
  
  - Setup Steps:
    1. Create context: `const UserContext = React.createContext();`
    2. Provide values: `<UserContext.Provider value={userData}>...children</UserContext.Provider>`
    3. Consume values: `const user = useContext(UserContext);`

* Global State Alternatives:
  - Context API: Native, good for low-frequency updates (e.g. themes, auth profiles). Can 
    cause performance issues (unwanted re-renders) on high-frequency changes.
  - Zustand / Redux Toolkit: External state engines utilizing centralized selectors to prevent 
    unnecessary re-renders on components.


7. FORMS IN REACT [Importance: 7.5/10]
--------------------------------------------------------------------------------
* Controlled Components:
  Form input values are driven strictly by React state.
  ```jsx
  const [name, setName] = useState('');
  return (
    <input value={name} onChange={(e) => setName(e.target.value)} />
  );
  ```

* Uncontrolled Components:
  Form data is handled by the DOM itself. Access values using a reference (`useRef`).
  ```jsx
  const inputRef = useRef(null);
  const handleSubmit = () => console.log(inputRef.current.value);
  return <input ref={inputRef} />;
  ```


8. PERFORMANCE OPTIMIZATION [Importance: 8.5/10]
--------------------------------------------------------------------------------
* `React.memo`:
  A higher-order component that shallowly compares props and skips re-rendering if props 
  are unchanged.

* `useMemo`:
  Caches the computed result of an expensive calculation between renders.
  ```javascript
  const expensiveVal = useMemo(() => computeHeavyData(list), [list]);
  ```

* `useCallback`:
  Caches the function definition itself between renders to prevent children from 
  re-rendering when passed callbacks as props.
  ```javascript
  const handleToggle = useCallback(() => setIsActive(prev => !prev), []);
  ```

* Virtual Lists (Virtualization):
  Only rendering elements currently visible in the viewport. Prevents browser sluggishness 
  when handling thousands of list rows (using libraries like `react-window`).


9. ROUTING (REACT ROUTER) [Importance: 7/10]
--------------------------------------------------------------------------------
* Basic Setup (React Router v6+):
  ```jsx
  import { createBrowserRouter, RouterProvider } from 'react-router-dom';

  const router = createBrowserRouter([
    { path: '/', element: <Home /> },
    { path: '/profile/:id', element: <Profile /> },
    { path: '*', element: <NotFound /> }
  ]);

  function App() {
    return <RouterProvider router={router} />;
  }
  ```

* Navigation:
  - Declared Link: `<Link to="/login">Login</Link>`
  - Programmatic:
    ```javascript
    const navigate = useNavigate();
    const handleLogin = () => navigate('/dashboard');
    ```


10. CODE SPLITTING & SUSPENSE [Importance: 7.5/10]
--------------------------------------------------------------------------------
* Dynamic Import (`React.lazy`):
  Allows components to be loaded lazily in separate chunks only when rendered.
  ```javascript
  const LazyProfile = React.lazy(() => import('./Profile'));
  ```

* Suspense:
  Wraps lazy components to display a fallback UI loader while files are being downloaded.
  ```jsx
  <Suspense fallback={<div>Loading component...</div>}>
    <LazyProfile />
  </Suspense>
  ```


11. EVENT HANDLING & SYNTHETIC EVENTS [Importance: 7/10]
--------------------------------------------------------------------------------
* SyntheticEvent:
  React wraps native browser events in a cross-browser helper called `SyntheticEvent` to 
  ensure consistent properties across Safari, Chrome, and Firefox.
  - Event Pooling (Historical): React pooled events for performance before React 17. 
    In modern React, event properties are fully retained.

* Debouncing Input changes:
  ```javascript
  import { debounce } from 'lodash';

  const debouncedSearch = useCallback(
    debounce((query) => apiSearch(query), 500),
    []
  );
  ```


12. MODERN REACT ADDITIONS (REACT 18 & 19) [Importance: 8/10]
--------------------------------------------------------------------------------
* Server Components (RSC vs Client Components):
  - React Server Components (RSC): Rendered on the server. Zero impact on client bundle size 
    because dependencies are not sent to the client. Excellent for initial loads and SEO.
  - Client Components (indicated by `'use client'` directive): Executed in the browser. 
    Can use state, effects, hooks, and browser APIs.

* Transition & Non-Blocking Updates:
  - `useTransition`: Splits updates into urgent (typing, clicking) and non-urgent transitions 
    (fetching lists). Prevents UI locks during rendering.
    ```javascript
    const [isPending, startTransition] = useTransition();
    
    startTransition(() => {
      setFilterQuery(query); // Runs asynchronously without blocking UI typing
    });
    ```

* React 19 Action Hooks:
  - `useActionState`: Handles async actions, loading states, and error handling seamlessly.
  - `useFormStatus`: Provides status information (like `pending`) of the parent `<form>` 
    inside child components.
