1. WHAT ARE WEBSOCKETS?
--------------------------------------------------------------------------------
* Definition:
  WebSockets is a network protocol that enables continuous, real-time, bidirectional 
  (full-duplex) communication between a client (browser) and a server over a single, 
  long-lived TCP connection.

* Core Characteristics:
  - Persistent Connection: Unlike standard HTTP where connections close after each request-response,
    the TCP socket remains open.
  - Bidirectional Data Flow: Both client and server can send messages at any time.
  - Protocols: `ws://` (non-secure over Port 80) and `wss://` (secure encrypted over Port 443).
  - Low Overhead: Bypasses expensive HTTP headers once established, transmitting packets 
    with minimal framing data.


2. REAL-TIME COMMUNICATIONS PROTOCOLS COMPARISON
--------------------------------------------------------------------------------
--------------------------------------------------------------------------------

--------------------------------------------------------------------------------
Protocol      | HTTP/1.1 or HTTP/2| HTTP/1.1 or HTTP/2| WS / WSS          | HTTP (text/event-stream)
Connection    | Short-lived      | Held open by server| Persistent        | Persistent (uni-directional)
Data Flow     | Uni-directional  | Uni-directional  | Full-Duplex       | Server-to-Client only
Latency       | Highest          | Medium           | Lowest (Real-Time)| Low (Real-Time)

Reconnection  | Automatic        | Automatic        | Manual logic      | Automatic built-in
Use Cases     | Inactive widgets | Chat fallback    | Gaming, Live chats| Stock feeds, live logs

3. THE WEBSOCKET HANDSHAKE PROTOCOL
--------------------------------------------------------------------------------
WebSockets start their lifecycle as a standard HTTP/1.1 request. The client requests to 
"Upgrade" the protocol:

1. **Client Upgrade Request**:
   ```http
   GET /chat HTTP/1.1
   Host: server.example.com
   Upgrade: websocket
   Connection: Upgrade
   Sec-WebSocket-Key: dGhlIHNhbXBsZSBub25jZQ==
   Sec-WebSocket-Version: 13
   ```
2. **Server Handshake Response**:
   The server agrees by responding with status **101 Switching Protocols**:
   ```http
   HTTP/1.1 101 Switching Protocols
   Upgrade: websocket
   Connection: Upgrade
   Sec-WebSocket-Accept: s3pPLMBiTxaQ9kYGzzhZRbK+xOo=
   ```


4. WEBSOCKET CONNECTION LIFECYCLE & READYSTATES
--------------------------------------------------------------------------------
The WebSocket object exposes a `readyState` property representing the connection status:
- `0` (`CONNECTING`): Connection is not yet open.
- `1` (`OPEN`): Connection is open and ready to communicate.
- `2` (`CLOSING`): Connection is in the process of closing.
- `3` (`CLOSED`): Connection is closed or could not be opened.


5. MODERN ALTERNATIVES & COMPLEMENTARY TECHNOLOGIES
--------------------------------------------------------------------------------
- **Server-Sent Events (SSE)**: Standard HTTP-based push notifications from server to client.
  Very simple to implement but strictly unidirectional.
- **WebRTC**: Real-Time Communication protocol designed for direct Peer-to-Peer (P2P) 
  audio, video, and data streaming. High performance but highly complex routing (requires STUN/TURN servers).
- **GraphQL Subscriptions**: A higher-level state-sync wrapper built on top of WebSockets, 
  ideal for subscribing to query responses in GraphQL architectures.
- **Web Push API**: Standard browser system used to deliver notifications from servers to 
  clients even when the application tab is closed (orchestrated via Service Workers).


6. ANGULAR WEBSOCKET INTEGRATION
--------------------------------------------------------------------------------
* Robust Angular Service (`websocket.service.ts`):
  Using RxJS `webSocket` subject is the cleanest way to integrate sockets into Angular's 
  reactive framework as it handles observables, automatic serialization, and clean tear-downs.

  ```typescript
  import { Injectable } from '@angular/core';
  import { webSocket, WebSocketSubject } from 'rxjs/webSocket';
  import { Observable, EMPTY } from 'rxjs';
  import { catchError, retry, delay } from 'rxjs/operators';

  @Injectable({
    providedIn: 'root'
  })
  export class WebsocketService {
    private socket$!: WebSocketSubject<any>;
    private readonly WS_URL = 'wss://api.example.com/chat';

    public connect(): Observable<any> {
      if (!this.socket$ || this.socket$.closed) {
        this.socket$ = webSocket({
          url: this.WS_URL,
          // Automatically close connection when there are no subscribers
          closingObserver: {
            next: () => console.log('Closing WebSocket connection')
          }
        });
      }

      return this.socket$.asObservable().pipe(
        // Catch connection errors and retry with a 3-second delay
        catchError(error => {
          console.error('WebSocket Error:', error);
          return EMPTY;
        }),
        retry({ delay: 3000 })
      );
    }

    public sendMessage(msg: any): void {
      if (this.socket$) {
        this.socket$.next(msg);
      }
    }

    public close(): void {
      if (this.socket$) {
        this.socket$.complete();
      }
    }
  }
  ```

* Component Integration (`chat.component.ts`):
  ```typescript
  import { Component, OnInit, OnDestroy } from '@angular/core';
  import { Subscription } from 'rxjs';
  import { WebsocketService } from './websocket.service';

  @Component({
    selector: 'app-chat',
    template: `
      <div *ngFor="let msg of messages">
        <strong>{{ msg.username }}:</strong> {{ msg.text }}
      </div>
      <input [(ngModel)]="messageText" placeholder="Type a message..." />
      <button (click)="send()">Send</button>
    `
  })
  export class ChatComponent implements OnInit, OnDestroy {
    messages: any[] = [];
    messageText: string = '';
    username: string = 'User1';
    private wsSubscription!: Subscription;

    constructor(private wsService: WebsocketService) {}

    ngOnInit(): void {
      this.wsSubscription = this.wsService.connect().subscribe({
        next: (data) => {
          if (data.type === 'init') {
            this.messages = data.messages;
          } else {
            this.messages.push(data);
          }
        },
        error: (err) => console.error('Component received WS error:', err)
      });
    }

    send(): void {
      if (!this.messageText.trim()) return;
      this.wsService.sendMessage({
        username: this.username,
        text: this.messageText
      });
      this.messageText = '';
    }

    ngOnDestroy(): void {
      // Unsubscribe to release network port / close connection
      if (this.wsSubscription) {
        this.wsSubscription.unsubscribe();
      }
    }
  }
  ```
