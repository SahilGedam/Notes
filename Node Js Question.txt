                    NODE.JS & EXPRESS INTERVIEW QUESTIONS

PART 1: NODE.JS CORE ARCHITECTURE & BASICS
--------------------------------------------------------------------------------
* What is Node.js? Where can you use it?
  Node.js is an open-source, cross-platform JavaScript runtime environment and library
  used to execute JavaScript code outside the client's browser (server-side).
  - Use cases: Data-intensive I/O applications (video streaming sites), real-time apps
    (chat, collaboration tools), network applications, and microservices architectures.

* Why use Node.js? What are its primary advantages?
  Node.js makes building scalable network programs easy by offering:
  - High performance & execution speed (built on V8).
  - Non-blocking asynchronous I/O execution.
  - Great concurrency using a single-threaded architecture.
  - Unified language (JavaScript) on both frontend and backend.

* What is the difference between JavaScript and Node.js?
  - JavaScript: A high-level, lightweight programming/scripting language. Runs on browser
    clients using engines like V8, SpiderMonkey, or JavaScriptCore.
  - Node.js: A runtime environment that wraps Chrome's V8 engine, providing server-side APIs
    (OS, filesystem, network access) to execute JavaScript code on a server.

* Why is Node.js single-threaded? How does it handle concurrency?
  - By handling async processing on a single thread under typical web loads, Node.js reduces
    the overhead of thread switching and memory allocation per request.
  - Concurrency is managed via the **Single-Threaded Event Loop Model**. Instead of waiting
    for blocking processes (like file access or db queries), Node delegates tasks to the OS
    kernel or Libuv thread pool and continues executing other requests. When tasks finish,
    their callbacks are pushed to the event queue to be executed by the single thread.

* Differentiate between Synchronous (Blocking) and Asynchronous (Non-Blocking) APIs.
  - Synchronous: Blocks execution of subsequent lines of code until the current function
    returns its result (e.g. `fs.readFileSync`).
  - Asynchronous: Initiates the operation, immediately registers a callback, and moves on to
    the next lines of code without blocking. The result is handled later (e.g. `fs.readFile`).

* What is a callback function? What is callback hell?
  - Callback: A function passed as an argument to another function, executed when an
    asynchronous task finishes.
  - Callback Hell: A situation where multiple nested callbacks depend on each other, forming
    deeply indented, unreadable code blocks (the "pyramid of doom").

* What are the benefits of using Promises over callbacks? How do we avoid callback hell?
  - Promises provide clean control flow, native chained error handling, and low coupling.
  - Methods to avoid callback hell:
    1. Using `async/await` (modern syntactic sugar).
    2. Using Promise chains (`.then().catch()`).
    3. Using Generators/yield (older approach).

* How would you define the term I/O?
  I/O (Input/Output) refers to the transfer of data to or from a medium (e.g. physical devices,
  networks, file systems, databases) relative to the runtime memory.

* What is NPM (Node Package Manager) and its core functionalities?
  NPM is the package manager for Node.js. It:
  - Hosts a global registry of open-source packages.
  - Provides a CLI utility to install packages, manage dependencies, and lock project versions.

* What are modules in Node.js? Explain `module.exports`.
  - Modules are reusable units of code containing functions and values. They are imported
    using the `require()` function (CommonJS syntax).
  - `module.exports` (or `exports`) is the special object exposed by a module. Whatever is
    assigned to `module.exports` is returned when another file imports it.

* What is the role of the `package.json` file?
  `package.json` is the metadata manifest file for a Node.js project. It lists project scripts,
  author information, dependencies, devDependencies, and the entry-point file name.

* Explain the role of the following built-in modules:
  - `url`: Utilities for URL resolution, formatting, and parsing.
  - `crypto`: Cryptographic utilities for hashing, encryption, decryption, and security keys.
  - `timers`: Global utilities (e.g. `setTimeout`, `setInterval`, `setImmediate`) for scheduling code.

* Differentiate between `process.nextTick()`, `setImmediate()`, and `setTimeout()`.
  - `process.nextTick()`: Schedules a callback to execute immediately *after the current operation
    completes*, before moving to the next event loop phase. It has the highest microtask priority.
  - `setImmediate()`: Schedules callbacks in the "Check" phase of the event loop.
  - `setTimeout()`: Holds execution until a specified delay (in milliseconds) expires.

* What is an `EventEmitter`?
  A core class in Node.js that enables publisher-subscriber communication. Objects extending
  `EventEmitter` emit named events and register listeners. When an event is triggered,
  all listener callbacks are invoked synchronously in order.

* What is a Buffer?
  A fixed-size memory chunk allocated outside the V8 heap, used to manipulate raw binary data
  directly (e.g. reading file bytes or network packets). Unlike arrays, buffers are not resizable.

* What are streams in Node.js? What are the four types?
  Streams are APIs used to read or write data chunk-by-chunk sequentially to prevent memory overload.
  - `Readable`: Read operations (e.g. `fs.createReadStream`).
  - `Writable`: Write operations (e.g. `fs.createWriteStream`).
  - `Duplex`: Readable & Writable (e.g. Web sockets).
  - `Transform`: Duplex stream that modifies data on the fly (e.g. `zlib` compression).

* What is the `fork` method in Node.js?
  A method in the `child_process` module used to spawn a new instance of the Node.js V8 engine
  to run a script. It creates an IPC (Inter-Process Communication) channel for communication
  between master and worker processes.

* What is REPL?
  REPL stands for Read-Eval-Print-Loop. It is the interactive command line environment used to
  run raw JavaScript statements and debug code.

* What is the difference between Node.js and AJAX?
  - AJAX (Asynchronous JavaScript and XML): A client-side browser technology used to make
    asynchronous requests to a server without refreshing the page.
  - Node.js: A server-side runtime used to build the web servers that process client requests.


PART 2: EXPRESS.JS FRAMEWORK & APIs
--------------------------------------------------------------------------------
* What is Express.js? How does it differ from raw Node.js?
  - Express.js: A lightweight, minimalist web application framework built on top of Node.js.
  - Difference: Node.js provides the raw network capabilities (`http` module), while Express
    simplifies routing, request parsing, headers setup, and adds middleware pipelines.

* Differentiate between a traditional web server and an Express.js server.
  - Traditional Server: Often requires manual code parsing, nested URL conditional statements
    to parse endpoints, and direct stream management.
  - Express.js Server: Runs on top of Node.js, providing out-of-the-box routing, request-response
    utilities, and structured middleware configurations.

* What are the main features of Express.js?
  - Robust Routing system.
  - Middleware framework wrapper.
  - High performance & simplicity.
  - Simple static file hosting (`express.static`).
  - Native integration with template views.

* Explain the common directory structure of an Express.js application.
  - Entry point (`app.js` / `server.js`): Setup, db connections, middleware registration.
  - `/routes`: Router handlers mapping HTTP endpoints.
  - `/controllers`: Request-response handlers containing business logic.
  - `/models`: Database schemas.
  - `/middleware`: Custom middleware handlers.
  - `/public`: Static public assets.

* How do you create a simple Express application? Explain `req` and `res` objects.
  ```javascript
  const express = require('express');
  const app = express();
  
  app.get('/', (req, res) => {
    res.send('Hello World');
  });
  app.listen(3000);
  ```
  - `req` (Request): Represents the HTTP request, containing headers, parameters (`req.params`),
    queries (`req.query`), cookies, and body payload.
  - `res` (Response): Represents the HTTP response sent back to the client.

* What is the `express.Router()` function?
  A function used to create a modular, mountable route handler. It allows grouping routes
  together in separate files to keep routing organized.

* Differentiate between `app.route()` and `app.use()`.
  - `app.route()`: Declares chained HTTP method handlers (GET, POST, PUT, etc.) for a single
    URI path.
  - `app.use()`: Mounts middleware functions globally or on specific path prefixes.

* What arguments are available to route handlers? What is the `next()` function?
  - Arguments: `req` (request), `res` (response), and `next` (callback).
  - `next()`: Pass control to the next middleware in the execution chain. If it is omitted,
    the request will hang. Passing an argument to it (`next(err)`) triggers error-handling.

* What is middleware? What are the 5 types of middleware in Express?
  Middleware functions run during the request-response cycle before endpoint logic is reached.
  - Types:
    1. Application-level: Bound to `app` instance using `app.use()`.
    2. Router-level: Bound to `express.Router()` instance.
    3. Error-handling: Takes 4 arguments: `(err, req, res, next)`.
    4. Built-in: Handled natively by Express (`express.json()`, `express.static()`).
    5. Third-party: Installed via npm (`cors`, `helmet`, `morgan`).

* What is the difference between `res.send()` and `res.json()`?
  - `res.send()`: A versatile method that automatically sets headers depending on the passed
    data type (Buffer, string, object, array).
  - `res.json()`: Formats javascript object payloads to JSON and sets the HTTP response header
    to `Content-Type: application/json` explicitly.

* What is dynamic routing? How is it implemented?
  Routes containing variable paths. Implemented using colons (e.g. `/users/:id` -> the value is
  retrieved via `req.params.id`).

* What is the purpose of `response.cookie()`?
  Used to attach cookie data to client browser responses:
  `res.cookie('session_id', '12345', { httpOnly: true });`

* Why should you separate the Express app definition and server listening setup?
  Separating the Express app configuration (routes, middleware definitions) from the listening server:
  - Promotes modularity.
  - Facilitates testing (allows running supertest scripts without binding to local ports).
  - Supports multiple deployment environments.


PART 3: ENVIRONMENT, SECURITY & BEST PRACTICES
--------------------------------------------------------------------------------
* What is a `.env` file?
  Used to store system configuration variables and secret keys (database URIs, ports, API tokens)
  locally, keeping sensitive settings out of version control.

* What is JSON Web Token (JWT)?
  A stateless authentication token comprising Header, Payload, and Signature. Assigned on login
  and attached to incoming request headers to authorize access.

* What is ESLint?
  A static analysis tool that parses JavaScript/ECMAScript syntax to detect formatting errors,
  dead code, and enforce style guides.

* What is CORS? Under what circumstances do requests fail?
  - CORS (Cross-Origin Resource Sharing): A security feature implemented by browsers that prevents
    scripted requests from accessing resources on a different domain unless allowed by the server.
  - Failure cases: Mismatched origins, lack of headers like `Access-Control-Allow-Origin`,
    unsupported HTTP methods, or requesting credentials without server approval.
