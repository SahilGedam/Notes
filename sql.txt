1. DATABASE CLASSIFICATIONS & SQL COMMAND TYPES [Importance: 8.5/10]
--------------------------------------------------------------------------------
* DBMS vs. RDBMS:
  - DBMS (Database Management System): Software used to store, manage, and retrieve data 
    stored as files (flat structures).
  - RDBMS (Relational DBMS): A database system based on the relational model (tables with 
    predefined columns and relationships). Enforces referential integrity.

* SQL Statement Classifications:
  - DDL (Data Definition Language): Defines the schema structure.
    Commands: `CREATE`, `ALTER`, `DROP`, `TRUNCATE`, `RENAME`.
  - DML (Data Manipulation Language): Modifies record data.
    Commands: `INSERT`, `UPDATE`, `DELETE`.
  - DQL (Data Query Language): Retrieves data from the database.
    Command: `SELECT`.
  - DCL (Data Control Language): Manages access rights and permissions.
    Commands: `GRANT`, `REVOKE`.
  - TCL (Transaction Control Language): Manages transactional changes.
    Commands: `COMMIT`, `ROLLBACK`, `SAVEPOINT`.


2. DATABASE CONSTRAINTS [Importance: 8.5/10]
--------------------------------------------------------------------------------
Constraints enforce data integrity rules on columns:
- `NOT NULL`: Restricts null values from being inserted into the column.
- `UNIQUE`: Guarantees that all values in the column are distinct.
- `CHECK`: Evaluates inserted values against a boolean condition (e.g. `CHECK (age >= 18)`).
- `DEFAULT`: Automatically fills a fallback value if no value is specified.
- `PRIMARY KEY`: Uniquely identifies each record in a table. It is implicitly `UNIQUE` and 
  `NOT NULL`. A table can have only one primary key.
- `FOREIGN KEY`: Establishes a link to a primary key in another table, enforcing referential 
  integrity (prevents orphaned records).


3. QUERY EXECUTION ORDER (FWGHSOL) [Importance: 9.5/10]
--------------------------------------------------------------------------------
Although queries are written starting with `SELECT`, the database engine executes the clauses 
in the following logical order:

1. **FROM / JOIN**: Database locates the target tables and joins them.
2. **WHERE**: Filters row records based on conditional checks. (Cannot filter aggregate values).
3. **GROUP BY**: Groups filtered rows based on column values.
4. **HAVING**: Filters grouped summaries (supports aggregate checks like `COUNT(*) > 5`).
5. **SELECT**: Evaluates columns, aliases, and expressions to return.
6. **DISTINCT**: Removes duplicate row results.
7. **ORDER BY**: Sorts rows (default `ASC` / `DESC`).
8. **LIMIT / OFFSET**: Restricts the size of the final returned rows.


4. JOINS & SET OPERATIONS [Importance: 9.5/10]
--------------------------------------------------------------------------------
* JOINS (Combining tables horizontally based on columns):
  - `INNER JOIN`: Returns records that have matching keys in both tables.
  - `LEFT JOIN` (Outer): Returns all records from the left table, and matching records 
    from the right table (unmatched right fields return `NULL`).
  - `RIGHT JOIN` (Outer): Returns all records from the right table, and matching records 
    from the left table.
  - `FULL OUTER JOIN`: Returns all records when there is a match in either left or right 
    tables (unmatched fields return `NULL`).
  - `SELF JOIN`: A regular join where a table is joined with itself (requires table aliasing).
    ```sql
    SELECT e1.name AS Employee, e2.name AS Manager
    FROM Employees e1
    JOIN Employees e2 ON e1.manager_id = e2.id;
    ```

* SET OPERATIONS (Combining query results vertically - requires matching column counts & types):
  - `UNION`: Combines results, removing duplicates.
  - `UNION ALL`: Combines results, retaining duplicates (faster than `UNION` as no sorting/filtering is done).
  - `INTERSECT`: Returns only rows that exist in the results of both queries.
  - `MINUS` / `EXCEPT`: Returns rows from the first query that do not exist in the second query.


5. NORMALIZATION & DENORMALIZATION [Importance: 9/10]
--------------------------------------------------------------------------------
* Normalization:
  The systematic process of organizing data attributes to avoid redundancy, anomalies, 
  and data duplication.
  - **1NF (First Normal Form)**: Atomic values only. No repeating groups or multi-valued attributes.
  - **2NF (Second Normal Form)**: Must be in 1NF. All non-prime attributes must be fully 
    dependent on the entire primary key (no partial dependencies on composite keys).
  - **3NF (Third Normal Form)**: Must be in 2NF. No transitive dependencies (non-key columns 
    cannot depend on other non-key columns).
  - **BCNF (Boyce-Codd Normal Form)**: Stronger version of 3NF. For every functional dependency 
    `X -> Y`, `X` must be a super key.

* Denormalization:
  The process of intentionally adding redundant data to database tables to speed up complex read 
  queries (bypasses expensive joins) at the cost of slower writes and higher storage.


6. DATABASE OBJECTS [Importance: 9/10]
--------------------------------------------------------------------------------
* Index:
  A data structure (often B-Tree or Hash) used to speed up queries.
  - `Clustered Index`: Determines the physical storage order of rows in the table. Only one per table.
  - `Non-Clustered Index`: Separate lookup tree pointing back to the physical row data. Multiple allowed.

* View:
  A virtual table containing columns and rows derived from a saved query. Does not store physical data.

* Stored Procedure:
  Precompiled SQL code saved in the database. Can accept inputs/outputs, handle transaction logic, 
  and is executed manually.

* Trigger:
  A database listener object that automatically executes a block of code in response to specific 
  events (e.g., `BEFORE INSERT`, `AFTER UPDATE`, or `DELETE`) on a table.


7. TRANSACTIONS & ACID PROPERTIES [Importance: 10/10]
--------------------------------------------------------------------------------
* ACID Properties:
  - **Atomicity**: The entire transaction completes successfully, or the database rollbacks 
    completely (All or Nothing).
  - **Consistency**: Enforces that all database rules (constraints, keys) are satisfied before 
    and after a transaction.
  - **Isolation**: Concurrent transactions run independently without interfering with one another.
  - **Durability**: Once a transaction commits, changes are written to non-volatile storage 
    and survive system failures.

* Isolation Levels & Anomalies:
  - **Anomalies**:
    - *Dirty Read*: Transaction reads uncommitted updates from another transaction.
    - *Non-Repeatable Read*: Re-reading the same row returns different column values because 
      another transaction committed an update in between.
    - *Phantom Read*: Re-running a query returns new matching rows because another transaction 
      inserted them.
  
  - **Levels**:
    -----------------------------------------------------------------------------
    Isolation Level  | Dirty Read? | Non-Repeatable Read? | Phantom Read?
    -----------------------------------------------------------------------------
    Read Uncommitted | Yes         | Yes                  | Yes
    Read Committed   | No          | Yes                  | Yes
    Repeatable Read  | No          | No                   | Yes
    Serializable     | No          | No                   | No
    -----------------------------------------------------------------------------


8. DATA REMOVAL OPERATIONS: DELETE vs. TRUNCATE vs. DROP
--------------------------------------------------------------------------------
--------------------------------------------------------------------------------
Feature      | DELETE                         | TRUNCATE             | DROP
--------------------------------------------------------------------------------
Type         | DML                            | DDL                  | DDL
Scope        | Selected rows (using WHERE)    | All rows             | Entire table structure
Rollback?    | Yes (writes to transaction log)| Yes (in PG/MSSQL); No (in MySQL/Oracle)| No
Speed        | Slow (logs row-by-row)         | Fast                 | Fast
Resets ID?   | No                             | Yes (Auto-increment) | Yes (Deletes table)
--------------------------------------------------------------------------------
