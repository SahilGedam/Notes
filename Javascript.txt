1. JAVASCRIPT BASICS & EXECUTION CONTEXT [Importance: 9/10]
--------------------------------------------------------------------------------
* What is JavaScript?
  JavaScript is a lightweight, cross-platform, single-threaded, and just-in-time 
  (JIT) compiled programming language. It is traditionally known as the scripting 
  language for webpages, but it is now widely used in many non-browser environments 
  (e.g., Node.js).
  - It is weakly (dynamically) typed.
  - It supports both imperative and declarative paradigms.
  - Contains a standard library of built-in objects (Array, Date, Math) and core 
    language elements (operators, control structures, statements).

* Strict Mode:
  Introduced in ECMAScript 5, strict mode is a way to opt into a restricted variant 
  of JavaScript that throws errors for silent failure cases.
  - Enabled by adding the `'use strict'` directive at the top of a script or function.
  - Characteristics:
    - Prevents duplicate parameter names (e.g. `function x(a, a) {}` is a SyntaxError).
    - Restricts naming variables/parameters after reserved words (like `eval`, `arguments`).
    - Prevents implicit creation of global variables (assigning to an undeclared 
      variable throws a `ReferenceError`).
    - In strict mode, `this` in global function calls evaluates to `undefined` 
      instead of the `window`/global object.

* Transpilers & Polyfills:
  Tools used to ensure modern JavaScript features run on older browser engines.
  - Transpilers (e.g. Babel): Source-to-source compilers that translate new syntax 
    into equivalent older syntax.
    - Example: Transpiling the nullish coalescing operator:
      ```javascript
      // Modern input:
      height = height ?? 100;
      
      // Transpiled output:
      height = (height !== undefined && height !== null) ? height : 100;
      ```
  - Polyfills (e.g. core-js): Scripts that add missing built-in functions/methods.
    - Example: Fills in `Math.trunc(1.23)` for engines lacking it.

* Global Object:
  Provides variables and functions available anywhere in the scope.
  - Browser: `window`
  - Node.js: `global`
  - Standardized (across all environments): `globalThis`

* Eval Function:
  Allows execution of code represented as a string:
  ```javascript
  let result = eval('2 + 2'); // Returns 4
  ```
  Note: Avoid using `eval` in production as it poses severe security risks and 
  degrades execution performance.

* Types of Errors in JavaScript:
  - Syntax Error: Mistakes or spelling/structure problems in the code that cause the 
    program to fail parsing and not execute at all (or stop running).
  - Runtime Error: Valid syntax, but execution fails at that point (e.g., accessing a 
    property on `null`, calling a non-function). The program starts running but throws 
    an exception mid-execution.
  - Logical Error: Program logic is incorrect, but the syntax is valid. The application 
    runs without throwing browser errors, but produces incorrect results. These are often 
    harder to debug.


2. DATA TYPES & COERCION [Importance: 8.5/10]
--------------------------------------------------------------------------------
* Stack Memory vs. Heap Memory:
  - Stack Memory:
    - Used for static memory allocation and execution context tracking.
    - Stores local variables of primitive types and reference pointers to objects in the heap.
    - Access is fast, managed automatically by the call stack structure (LIFO).
  - Heap Memory:
    - Used for dynamic memory allocation.
    - Stores objects (including arrays, functions, dates, etc.).
    - Access is slower than the stack, and space is managed by the JavaScript Garbage Collector.

* Primitive Types vs. Reference Types:
  - Primitive Types:
    - String, Number, Boolean, BigInt, Undefined, Null, Symbol.
    - Stored directly in the Stack memory because they have a fixed size.
    - Immutable: The value itself cannot be changed. Reassigning a variable creates a new value in memory.
  - Reference Types (Objects):
    - Plain Objects, Arrays, Functions, Dates, RegExp, etc.
    - The actual data is stored in the Heap memory because its size is dynamic.
    - A reference pointer (address) to that location in the Heap is stored in the Stack variable.
    - Mutable: Properties and elements can be changed without changing the reference address.

* Pass by Value vs. Pass by Reference (Call by Sharing):
  - JavaScript is strictly **pass-by-value**. However, how that value is interpreted depends on the data type:
  - Primitives (Pass by Value):
    - When passing a primitive to a function or assigning to another variable, a copy of the actual value is made.
    - Changes to the copy do not affect the original variable.
  - Objects (Pass by Value of Reference / Call by Sharing):
    - When passing an object, a copy of the reference pointer (address) is passed by value.
    - As a result, both variables point to the same object in Heap memory. Mutating a property inside a function affects the original object.
    - Crucially: Reassigning the parameter inside the function points it to a new memory address, breaking the link and leaving the original object unchanged.
    - Example:
      ```javascript
      function update(num, obj) {
        num = 100; // Copy of primitive value updated, original remains 10
        obj.name = "Updated"; // Mutates heap object through shared reference
        obj = { name: "Reassigned" }; // Reassigns local parameter reference, original unchanged
      }
      let x = 10;
      let y = { name: "Original" };
      update(x, y);
      console.log(x); // 10
      console.log(y.name); // "Updated" (not "Reassigned")
      ```

* `typeof` Operator Output Examples:
  ```javascript
  typeof "John Doe" // "string"
  typeof 3.14       // "number"
  typeof true       // "boolean"
  typeof 42n        // "bigint"
  typeof undefined  // "undefined"
  typeof null       // "object" (acknowledged legacy bug in JS)
  typeof Symbol()   // "symbol"
  typeof {}         // "object"
  typeof function(){} // "function"
  ```

* Implicit Type Coercion:
  Automatic conversion of values from one data type to another during operations.
  - String Coercion: Triggered by the binary `+` operator when one operand is a string.
    ```javascript
    "3" + 4 // "34" (number 4 is coerced to "4")
    ```
  - Boolean Coercion: Occurs in logical checks (loops, `if` statements).
    - Falsy values: `false`, `0`, `-0`, `0n`, `""`, `null`, `undefined`, and `NaN`.
    - Everything else is truthy.
  - Equality Coercion: Occurs during loose equality comparisons (`==`).

* Loose (`==`) vs. Strict (`===`) Equality:
  - `==` (Loose): Compares values after performing type coercion.
    ```javascript
    "5" == 5 // true
    ```
  - `===` (Strict): Compares both values and types without coercion.
    ```javascript
    "5" === 5 // false
    ```
  - Classic Interview Question: Why does `[] == ![]` evaluate to `true`?
    1. `!` has higher precedence than `==`. So `![]` is evaluated first. Since arrays are truthy, `![]` becomes `false`.
    2. The expression is now `[] == false`.
    3. Loose equality (`==`) with a boolean coerces both to numbers: `false` becomes `0`.
    4. The expression is now `[] == 0`.
    5. The array `[]` is coerced to a primitive. `[].toString()` is `""` (empty string).
    6. The expression is now `"" == 0`.
    7. Empty string `""` coerced to a number is `0`.
    8. The expression becomes `0 == 0`, which is `true`.

* NaN Property (Not-a-Number):
  Represents an invalid or unrepresentable numerical calculation.
  - `typeof NaN` returns `"number"`.
  - Use global `isNaN()` or `Number.isNaN()` to check. Note that `isNaN()` coerces 
    its input to a number before checking, while `Number.isNaN()` does not.
    ```javascript
    isNaN("hello")        // true (coerces "hello" to NaN)
    Number.isNaN("hello") // false (type is string, not NaN number)
    ```

* Infinity:
  A numeric property representing positive infinity (greater than any number).
  `typeof Infinity` returns `"number"`.

* Symbols:
  A primitive type representing unique, immutable identifiers.
  ```javascript
  let id1 = Symbol("id");
  let id2 = Symbol("id");
  console.log(id1 === id2); // false (always unique)
  ```
  - Auto-conversion: Symbols do not auto-convert to string. Direct string output 
    throws a TypeError; call `.toString()` explicitly.
  - Object Properties: Symbols are skipped by `for..in` loops and `Object.keys()`, 
    but are copied by `Object.assign()`. To retrieve symbol-keyed properties, use 
    `Object.getOwnPropertySymbols(obj)`.
  - Global Registry: Use `Symbol.for(key)` to retrieve or create a global symbol. 
    `Symbol.keyFor(sym)` returns its key name.

* Object to Primitive Conversion (Hints):
  JavaScript uses three internal "hints" when converting objects to primitives:
  - `"string"`: Used when an operation expects a string (e.g. `alert(obj)`).
  - `"number"`: Used when math is performed (e.g. `+obj`, `obj1 - obj2`).
  - `"default"`: Used when the operator is unsure (e.g. binary `+` or loose equality `==`).


3. VARIABLES & SCOPING [Importance: 9/10]
--------------------------------------------------------------------------------
* Global Scope vs. Function Scope vs. Block Scope:
  - Global Scope:
    - Variables declared outside any function or curly braces `{}` are in the global scope. They are accessible from anywhere in the application.
    - Global variables declared with `var` are added as properties to the global object (`window` or `globalThis`). Variables declared with `let` and `const` are not.
  - Function (Local) Scope:
    - Variables declared inside a function (using `var`, `let`, or `const`) are scoped to that function. They cannot be accessed from outside the function.
  - Block Scope:
    - Variables declared inside curly braces `{}` (such as in an `if` block, `for` loop, or raw block) using `let` or `const` are block-scoped. They cannot be accessed outside that block.
    - Note: `var` is NOT block-scoped; it ignores block boundaries and leaks to the enclosing function or global scope.

* Difference between var, let, and const:
  | Feature | `var` | `let` | `const` |
  |---------|-------|-------|---------|
  | Scope | Function Scope | Block Scope | Block Scope |
  | Hoisting | Hoisted & initialized to `undefined` | Hoisted but uninitialized (TDZ) | Hoisted but uninitialized (TDZ) |
  | Reassignable | Yes | Yes | No |
  | Redeclarable | Yes (in same scope) | No | No |
  | Global Object Property | Yes (when global) | No | No |

* What is Hoisting?
  - Hoisting is the JavaScript engine's behavior of moving declarations to the top of their containing scope during the compilation phase, before execution.
  - Value initializations are NOT hoisted; only the declarations are.
  - Function Declarations: Fully hoisted (both the name and body are loaded), allowing you to call the function before defining it in the code.
  - `var` variables: Hoisted and initialized to `undefined`.
  - `let` and `const` variables: Hoisted but left uninitialized, putting them in the Temporal Dead Zone (TDZ).
  - Classes and Imports: Hoisted but cannot be accessed until declared.

* What is Temporal Dead Zone (TDZ)?
  - The TDZ is the period between entering a block scope and the actual execution reaching the line where a variable is declared.
  - Attempting to access or reference a variable within its TDZ throws a `ReferenceError`.
  - Example:
    ```javascript
    {
      // TDZ for variable 'num' starts here
      // console.log(num); // ReferenceError: Cannot access 'num' before initialization
      let num = 10; // TDZ ends
      console.log(num); // 10
    }
    ```

* Lexical Scope vs. Scope Chain:
  - Lexical Scope (Static Scope):
    - JavaScript uses lexical scoping, meaning a variable's scope is determined by its position within the physical source code (at compile time), not where or how the function is called (dynamic).
    - Inner functions have access to variables declared in their outer nesting scopes.
  - Scope Chain:
    - The hierarchical chain of accessible scopes determined lexically.
    - When a variable is referenced, the JavaScript engine tries to find it in the current local scope. If not found, it traverses upward to the outer enclosing scope, continuing until it reaches the global scope. If the variable is still not found, it throws a `ReferenceError`.
    - Example:
      ```javascript
      const globalVar = "global";
      function outer() {
        const outerVar = "outer";
        function inner() {
          const innerVar = "inner";
          console.log(innerVar, outerVar, globalVar); // Resolves up the scope chain
        }
        inner();
      }
      outer();
      ```

* Variable Shadowing:
  - Variable shadowing occurs when a variable declared within a specific scope (like a block or function) has the same name as a variable in an outer scope.
  - The inner variable "shadows" (blocks access to) the outer variable within that inner scope.
  - Illegal Shadowing: You cannot shadow a block-scoped variable (`let` or `const`) using `var` within the same block scope. It throws a `SyntaxError` (Identifier has already been declared).
  - Example:
    ```javascript
    let x = 10;
    if (true) {
      let x = 20; // Valid Shadowing
      console.log(x); // 20
    }
    console.log(x); // 10
    
    // Illegal Shadowing Example:
    /*
    let y = 10;
    if (true) {
      var y = 20; // SyntaxError: Identifier 'y' has already been declared
    }
    */
    ```


4. FUNCTIONS & CLOSURES [Importance: 10/10]
--------------------------------------------------------------------------------
* Function Declaration vs. Function Expression:
  - Function Declaration:
    - Declared using the `function` keyword as a standalone statement.
    - Fully Hoisted: Can be called before its definition in the code because the JS engine loads declarations into memory before execution.
    - Example:
      ```javascript
      greet(); // "Hello" (works due to hoisting)
      function greet() { return "Hello"; }
      ```
    - Scope: In strict mode, function declarations inside a block `{}` are block-scoped.
  - Function Expression:
    - Created when a function is defined within an expression (typically assigned to a variable).
    - Not Hoisted: The variable is hoisted (according to `var`, `let`, or `const` rules), but the function definition is not initialized until the execution reaches that line.
    - Example:
      ```javascript
      greetExpr(); // ReferenceError or TypeError
      const greetExpr = function() { return "Hello"; };
      ```

* Arrow Function vs. Normal Function:
  Arrow functions (introduced in ES6) and normal functions have key differences:
  1. `this` Binding:
     - Normal Function: Has a dynamic `this` context. The value of `this` is determined by how the function is invoked (the caller).
     - Arrow Function: Does not have its own `this`. It inherits `this` lexically from its enclosing parent scope.
  2. Constructor / `new` operator:
     - Normal Function: Can be used as a constructor function with `new` (except when it's an ES6 shorthand method). It has a prototype property.
     - Arrow Function: Cannot be used as a constructor (throws `TypeError: ... is not a constructor`) and does not have a `prototype` property.
  3. `arguments` Object:
     - Normal Function: Has access to the local `arguments` object containing all passed parameters.
     - Arrow Function: Does not have its own `arguments` object (inherits it from the outer lexical scope). Use Rest Parameters (`...args`) instead.
  4. Duplicate Parameter Names:
     - Normal Function: Permitted in non-strict mode (though deprecated).
     - Arrow Function: Never allowed, even in non-strict mode (throws a `SyntaxError`).
  5. Syntax & Return:
     - Arrow Function: Offers implicit returns for single-line expressions (e.g., `(x) => x * 2`). Multiline bodies with `{}` require an explicit `return`.

* First-Class Functions:
  - Definition: A programming language is said to have First-Class Functions when functions in that language are treated like any other variable.
  - Characteristics:
    1. Can be stored in a variable: `const myFunc = function() {};`
    2. Can be passed as an argument to another function (Callback): `addEventListener('click', myFunc);`
    3. Can be returned from another function (Closure / Higher-Order): `return function() {};`

* Anonymous Functions vs. Named Functions:
  - Anonymous Function:
    - A function without a name. Often used as immediate callbacks or in function expressions.
    - Example:
      ```javascript
      setTimeout(function() { console.log("Run"); }, 1000);
      ```
  - Named Function:
    - A function defined with a specific name identifier. Useful for recursion, debugging (appears in call stacks), and readability.
    - Named Function Expression (NFE): A function expression that contains a name. The name is local ONLY to the function's own body (useful for recursive calls inside the expression).
      ```javascript
      const factorial = function fact(n) {
        if (n <= 1) return 1;
        return n * fact(n - 1); // 'fact' is accessible here
      };
      // console.log(fact); // ReferenceError: fact is not defined
      ```

* Callback Functions:
  - Definition: A function passed into another function as an argument, which is then invoked inside the outer function to complete some routine or action.
  - Synchronous Callback: Executed immediately (e.g., `Array.prototype.forEach(callback)`).
  - Asynchronous Callback: Executed at a later time (e.g., inside `setTimeout` or after an API request resolves).
  - Example:
    ```javascript
    function processInput(callback) {
      const name = "User";
      callback(name);
    }
    processInput(name => console.log("Hello " + name));
    ```

* Higher-Order Functions (HOF):
  - Definition: A function that operates on other functions, either by taking one or more functions as arguments (callbacks) or by returning a function.
  - Examples: `map()`, `filter()`, `reduce()`, decorators, and curry generators.
  - Example:
    ```javascript
    const multiplier = (factor) => (num) => num * factor;
    const double = multiplier(2);
    console.log(double(5)); // 10
    ```

* Pure Functions:
  - Definition: A function that satisfies two conditions:
    1. Deterministic: Always returns the same output for the same set of arguments.
    2. No Side Effects: Does not modify external state (variables outside its scope, global state, files, databases) or mutate its input arguments.
  - Example:
    ```javascript
    // Pure:
    const add = (a, b) => a + b;
    
    // Impure (depends on external variable):
    let tax = 0.05;
    const calculateTotal = (subtotal) => subtotal * (1 + tax);
    
    // Impure (modifies external/input state):
    const addToArray = (arr, val) => {
      arr.push(val); // Mutates input argument
      return arr;
    };
    ```

* Generator Functions:
  - Definition: Special functions that can pause their execution and resume later, yielding multiple values over time.
  - Declared using the `function*` syntax and pause execution using the `yield` keyword.
  - Returns a Generator object that adheres to both the iterable and iterator protocols. Calling `.next()` returns `{ value: any, done: boolean }`.
  - Example:
    ```javascript
    function* counter() {
      yield 1;
      yield 2;
      return 3;
    }
    const gen = counter();
    console.log(gen.next()); // { value: 1, done: false }
    console.log(gen.next()); // { value: 2, done: false }
    console.log(gen.next()); // { value: 3, done: true }
    ```

* Async Functions:
  - Definition: Functions declared with the `async` keyword, allowing them to write asynchronous code using `await` in a synchronous-looking style.
  - Return Value: Always returns a Promise. If the function returns a non-promise value, it is automatically wrapped in a resolved promise.
  - `await` keyword: Can only be used inside async functions (or top-level modules). It pauses execution until the awaited promise settles.
  - Example:
    ```javascript
    async function fetchUserData() {
      try {
        const response = await fetch('/api/user');
        const data = await response.json();
        return data;
      } catch (error) {
        console.error("Failed to fetch:", error);
      }
    }
    ```

* Closures:
  - **What is a Closure? (The Interview Definition)**:
    A closure is the combination of a function bundled together with references to its surrounding state (its **lexical environment**). 
    In simple terms: **A closure allows an inner function to access variables from its outer (enclosing) scope, even after the outer function has finished executing.**

  - **How it Works (Under the Hood)**:
    - **Lexical Scoping**: JavaScript uses lexical scope, meaning the nesting of functions determines variable access. Inner functions have access to variables declared in their outer scopes.
    - **Preservation**: When an outer function returns an inner function, the inner function maintains a reference to the outer function's scope chain (lexical environment). JavaScript's garbage collector will not clean up those outer variables because they are still referenced by the inner function.

  - **Why do we use Closures? (Use Cases)**:
    1. **Data Encapsulation / Private Variables**: Creating variables that cannot be accessed directly from the outside, only through exposed methods.
    2. **State Maintenance**: Keeping track of state without polluting the global scope.
    3. **Function Factories**: Creating customized functions with pre-configured values.

  - **Practical Code Examples**:
    * **Example 1: Basic Closure**
      ```javascript
      function outerFunction(outerVariable) {
        return function innerFunction(innerVariable) {
          console.log(`Outer: ${outerVariable}, Inner: ${innerVariable}`);
        };
      }
      const myClosure = outerFunction("outside");
      myClosure("inside"); // Outputs: "Outer: outside, Inner: inside"
      ```

    * **Example 2: Data Encapsulation (Private State / Counter)**
      ```javascript
      function createCounter() {
        let count = 0; // Private variable, cannot be accessed directly from outside
        return {
          increment: function() { count++; return count; },
          decrement: function() { count--; return count; },
          getCount: function() { return count; }
        };
      }
      const counter = createCounter();
      console.log(counter.increment()); // 1
      console.log(counter.increment()); // 2
      console.log(counter.getCount());   // 2
      // console.log(counter.count);    // undefined (perfect encapsulation!)
      ```

  - **Classic Interview Puzzle: The `for` loop with `var` vs `let`**
    * *The Problem (using `var`)*:
      ```javascript
      for (var i = 1; i <= 3; i++) {
        setTimeout(function() {
          console.log(i); // Outputs: 4, 4, 4 (after 1 second)
        }, 1000);
      }
      // Why? 'var' has function/global scope. All setTimeouts reference the exact same 'i' variable, which equals 4 when the loop completes.
      ```
    * *The Solution using Closures (Classic)*:
      ```javascript
      for (var i = 1; i <= 3; i++) {
        (function(currentNum) {
          setTimeout(function() {
            console.log(currentNum); // Outputs: 1, 2, 3 (each execution has its own closure state)
          }, 1000);
        })(i);
      }
      ```
      ```
      for (let i = 1; i <= 3; i++) {
        setTimeout(function() {
          console.log(i); // Outputs: 1, 2, 3 (block-scoped 'let' creates a new binding for 'i' in each iteration)
        }, 1000);
      }
      ```

  - **Real-World Use Cases (Crucial for Interviews)**:
    * **Use Case 1: Debouncing (Performance Optimization)**
      * *What it is*: Limits the rate at which a function triggers. Extremely useful for window resizing, scroll events, or search input auto-complete.
      * *How Closure helps*: It stores the `timeoutId` reference in the outer scope, keeping it private and persistent across consecutive function calls.
      ```javascript
      function debounce(func, delay) {
        let timeoutId; // Persistent timer state stored via closure
        
        return function(...args) {
          // Clear any active timers to reset the waiting period
          clearTimeout(timeoutId);
          
          // Set a new timer
          timeoutId = setTimeout(() => {
            func.apply(this, args); // Invoke the actual function
          }, delay);
        };
      }
      
      const handleSearch = debounce((query) => {
        console.log("Searching for:", query);
      }, 300);
      
      // If user types 'a', then 'ab', then 'abc' within 300ms, 
      // only "Searching for: abc" will run once.
      ```

    * **Use Case 2: Memoization (Caching)**
      * *What it is*: Caches the output of expensive function calls based on inputs, avoiding redundant operations.
      * *How Closure helps*: It maintains a private `cache` object in memory that is only accessible by the returned function.
      ```javascript
      function memoize(fn) {
        const cache = {}; // Private cache object preserved in outer scope
        
        return function(...args) {
          const key = JSON.stringify(args); // Create unique cache key
          if (cache[key]) {
            console.log("Cache hit!");
            return cache[key];
          }
          console.log("Cache miss! Computing result...");
          const result = fn(...args);
          cache[key] = result;
          return result;
        };
      }
      
      const expensiveSquare = memoize(x => x * x);
      expensiveSquare(5); // Cache miss! Computes and returns 25
      expensiveSquare(5); // Cache hit! Returns 25 instantly without computing
      ```

* Currying:
  Transforming a function of `N` arguments into `N` sequential functions of single arguments.
  ```javascript
  // Target Call Signature: add(1)(2)(3)
  
  // Curried Implementation:
  function add(a) {
    return function(b) {
      return function(c) {
        return a + b + c;
      };
    };
  }

  console.log(add(1)(2)(3)); // 6
  ```

* The "this" Keyword:
  - Definition: The `this` keyword is a reference to the object that is currently executing the function. Its value is not static; it is determined at runtime based on how the function is invoked.
  - "this" inside Normal Functions:
    - Global Invocation: In non-strict mode, `this` refers to the global object (`window` in browsers, `global` in Node.js). In strict mode, `this` is `undefined`.
    - Method Invocation: When a function is called as a method of an object (e.g., `obj.method()`), `this` points to the object before the dot (the invoking object).
    - Constructor Invocation: Inside a constructor function or class constructor called with `new`, `this` points to the newly created object instance.
    - Event Handler Invocation: In DOM event listeners, `this` points to the HTML element that received the event.
  - "this" inside Arrow Functions:
    - Arrow functions do **not** have their own `this` binding. Instead, they capture the `this` value of the enclosing lexical context at the time they are defined.
    - Changing `this` context on an arrow function using `call()`, `apply()`, or `bind()` has no effect; the lexical `this` remains unchanged.
    - Example:
      ```javascript
      const obj = {
        name: "Alice",
        normalMethod: function() {
          console.log("Normal:", this.name); // "Alice" (points to obj)
        },
        arrowMethod: () => {
          console.log("Arrow:", this.name); // undefined (points to global/window context)
        }
      };
      ```

* Setting "this" Context (call, apply, bind):
  - **The Core Concept**: In JavaScript, the `this` keyword refers to the object that is executing the current function. However, the value of `this` is dynamically determined by how the function is called. `call`, `apply`, and `bind` are built-in methods on all JavaScript functions used to **explicitly set and control the `this` context**, regardless of how or where the function is executed.

  - **Interview-Friendly Definitions**:
    1. **`call()`**:
       - **What it does**: Invokes (runs) the function immediately and sets the `this` keyword to the provided object.
       - **Argument Passing**: Arguments are passed **individually** (separated by commas).
       - **Mnemonic / Memory Trick**: **C**all -> **C**omma-separated.
       - **Example**: `greet.call(person, "arg1", "arg2")`
    2. **`apply()`**:
       - **What it does**: Invokes (runs) the function immediately and sets the `this` keyword to the provided object.
       - **Argument Passing**: Arguments are passed as a **single array** (or array-like object).
       - **Mnemonic / Memory Trick**: **A**pply -> **A**rray.
       - **Example**: `greet.apply(person, ["arg1", "arg2"])`
    3. **`bind()`**:
       - **What it does**: Does **not** invoke the function immediately. Instead, it returns a **new copy of the function** with its `this` keyword permanently bound to the provided object, which can be executed later.
       - **Argument Passing**: Arguments can be passed either during binding (partial application) or when invoking the bound function later.
       - **Mnemonic / Memory Trick**: **B**ind -> **B**ound function (deferred execution).
       - **Example**: `const bound = greet.bind(person); bound("arg1", "arg2");`

  - Key Comparison Table:
    | Method | Invocation | Argument Format | Return Value | Common Use Case |
    |--------|------------|-----------------|--------------|-----------------|
    | `call()` | Immediate | Comma-separated list | Result of the function | Function borrowing (using a method from another object) |
    | `apply()` | Immediate | Single Array `[...]` | Result of the function | Working with APIs that expect arrays (e.g., `Math.max.apply(null, arr)`) |
    | `bind()` | Deferred | Comma-separated list | A new bound function | Event listeners, callbacks (preventing `this` loss in setTimeout/React) |

  - Example: Basic usage of call, apply, and bind:
    ```javascript
    const person = { name: "Sahil" };
    
    function greet(age, hobby) {
      console.log(`Hello, my name is ${this.name}. I am ${age} years old and love ${hobby}.`);
    }
    
    // 1. call: Executes immediately, args passed individually (separated by commas)
    greet.call(person, 28, "coding");
    // Output: Hello, my name is Sahil. I am 28 years old and love coding.
    
    // 2. apply: Executes immediately, args passed as an array
    greet.apply(person, [28, "coding"]);
    // Output: Hello, my name is Sahil. I am 28 years old and love coding.
    
    // 3. bind: Returns a new function to be called later
    const boundGreet = greet.bind(person, 28, "coding");
    boundGreet(); 
    // Output: Hello, my name is Sahil. I am 28 years old and love coding.
    ```

  - Scenario 1: Function Borrowing:
    An object can borrow a method from another object without copying it.
    ```javascript
    const user1 = {
      name: "Vivian",
      age: 45,
      getDetails: function(role) {
        return `${this.name} is ${this.age} years old and works as a ${role}.`;
      }
    };
    
    const user2 = {
      name: "Alice",
      age: 30
    };
    
    // user2 borrows getDetails method from user1:
    const details = user1.getDetails.call(user2, "Software Engineer");
    console.log(details); // "Alice is 30 years old and works as a Software Engineer."
    ```

  - Scenario 2: Partial Application / Function Currying via bind:
    Pre-specifying initial arguments of a function when creating a new bound function.
    ```javascript
    function multiply(a, b) {
      return a * b;
    }
    
    // Pre-bind 'a' parameter to 2 (creates a doubling function)
    const double = multiply.bind(null, 2); 
    console.log(double(5)); // 10 (2 * 5)
    
    // Pre-bind 'a' parameter to 3 (creates a tripling function)
    const triple = multiply.bind(null, 3);
    console.log(triple(5)); // 15 (3 * 5)
    ```


5. OBJECTS & MEMORY MODEL [Importance: 9/10]
--------------------------------------------------------------------------------
* Object Basics & Reflection API:
  - Object.keys(obj): Returns an array of a given object's own enumerable string-keyed property names.
    ```javascript
    const user = { name: "Sahil", age: 28 };
    console.log(Object.keys(user)); // ["name", "age"]
    ```
  - Object.values(obj): Returns an array of a given object's own enumerable string-keyed property values.
    ```javascript
    console.log(Object.values(user)); // ["Sahil", 28]
    ```
  - Object.entries(obj): Returns an array of a given object's own enumerable string-keyed property `[key, value]` pairs.
    ```javascript
    console.log(Object.entries(user)); // [["name", "Sahil"], ["age", 28]]
    ```
  - Object.prototype.hasOwnProperty(prop): Returns a boolean indicating whether the object has the specified property as its own property (as opposed to inheriting it).
    Note: Modern best practice is to use `Object.hasOwn(obj, prop)` because `hasOwnProperty` can be shadowed or unavailable on objects created with `Object.create(null)`.
    ```javascript
    const parent = { inheritedProp: true };
    const child = Object.create(parent);
    child.ownProp = true;
    console.log(child.hasOwnProperty('ownProp'));       // true
    console.log(child.hasOwnProperty('inheritedProp')); // false (inherited)
    ```

* Object.assign(target, ...sources):
  - Copies all enumerable own properties from one or more source objects to a target object. It returns the modified target object.
  - Used for merging objects or cloning objects (shallow copy).
  - Example:
    ```javascript
    const target = { a: 1 };
    const source1 = { b: 2 };
    const source2 = { c: 3 };
    Object.assign(target, source1, source2);
    console.log(target); // { a: 1, b: 2, c: 3 }
    ```

* Object Destructuring & Spread Operator:
  - Object Destructuring: Allows unpacking properties from objects into distinct variables.
    ```javascript
    const person = { name: "Vivian", age: 45, role: "Engineer" };
    const { name, age, ...rest } = person; // rest collects remaining keys
    console.log(name, age); // "Vivian" 45
    console.log(rest);      // { role: "Engineer" }
    ```
  - Spread Operator (`...`): Used to copy or merge own enumerable properties from one object into another.
    ```javascript
    const baseConfig = { host: "localhost", port: 80 };
    const userConfig = { ...baseConfig, port: 443, debug: true }; // Merges and overrides port
    console.log(userConfig); // { host: "localhost", port: 443, debug: true }
    ```

* Memory Model (Pass by Value / Sharing):
  JavaScript is strictly **pass-by-value**.
  - Primitives: The value is copied.
  - Objects: The reference (memory address) to the object is copied and passed by value.
    - Property mutations inside a function affect the outer object.
    - Reassigning the object parameter does not change the outer variable.
  ```javascript
  function modify(obj) {
    obj.prop = "mutated"; // Affects outer object
    obj = { newProp: 2 }; // Does not affect outer object (reassigned local copy of reference)
  }
  ```

* Object Equality Comparison:
  Two objects are equal only if they reference the same object in memory.
  ```javascript
  let a = {};
  let b = a;
  console.log(a === b); // true (same reference)
  
  let c = {};
  let d = {};
  console.log(c === d); // false (different references in heap)
  ```

* Shallow vs. Deep Copy:
  - Shallow Copy: Copies only the top-level keys. Nested objects are not duplicated; their reference pointers are shared.
    - Methods: Object spread `{ ...obj }` or `Object.assign({}, obj)`.
    - Mutating nested properties on the clone affects the original object.
  - Deep Copy: Recursively copies all properties, ensuring nested objects are fully duplicated.
    - Methods:
      1. Native modern API: `structuredClone(obj)`. Fully clones objects, Dates, Maps, Sets, Arrays, etc.
      2. Serialization fallback: `JSON.parse(JSON.stringify(obj))` (Note: Loses methods, symbols, undefined values, and converts Date objects to strings).
  - Comparison Example:
    ```javascript
    const original = { name: "Alice", address: { city: "New York" } };
    
    // Shallow copy:
    const shallow = { ...original };
    shallow.address.city = "Boston"; // Modifies original.address.city as well!
    
    // Deep copy:
    const deep = structuredClone(original);
    deep.address.city = "Chicago"; // Does NOT affect original
    ```

* Restricting Object Mutations (Object.freeze vs Object.seal):
  - Object.seal(obj):
    - Prevents adding new properties and deletes existing properties (sets `configurable: false` on all properties).
    - Crucially: Existing property **values** can still be modified if they are writable.
    - Check with: `Object.isSealed(obj)`.
  - Object.freeze(obj):
    - Maximum protection. Prevents adding, removing, and changing property values (sets both `configurable: false` and `writable: false`).
    - Crucially: The freeze is **shallow**. Nested objects can still be mutated unless recursively frozen.
    - Check with: `Object.isFrozen(obj)`.
  - Comparison Example:
    ```javascript
    const user = { name: "John", meta: { age: 30 } };
    
    // Object.seal Example:
    Object.seal(user);
    user.name = "Jane"; // Allowed
    delete user.name; // Silent failure (or TypeError in strict mode)
    user.newProp = "test"; // Silent failure (or TypeError in strict mode)
    
    // Object.freeze Example:
    const frozen = { name: "John", meta: { age: 30 } };
    Object.freeze(frozen);
    frozen.name = "Jane"; // Silent failure (or TypeError in strict mode)
    frozen.meta.age = 35; // Allowed! (nested mutation because freeze is shallow)
    ```

* Object Property Descriptors:
  You can control properties with fine-grained configuration using `Object.defineProperty(obj, prop, descriptor)`:
  - `writable`: If `true`, the value of the property can be changed.
  - `enumerable`: If `true`, the property shows up in `for..in` loops and `Object.keys()`.
  - `configurable`: If `true`, the property can be deleted and its descriptors modified.
  ```javascript
  let user = {};
  Object.defineProperty(user, "id", {
    value: 101,
    writable: false, // Cannot be modified
    enumerable: false, // Hidden in loops
    configurable: false // Cannot be deleted
  });
  ```

* Getters & Setters:
  Accessor properties inside object literals:
  ```javascript
  let obj = {
    get propName() { return this._val; },
    set propName(value) { this._val = value; }
  };
  ```

* Optional Chaining (`?.`):
  Stops evaluation and returns `undefined` if the operand before `?.` is `null` or 
  `undefined`.
  - Property access: `obj?.prop`
  - Dynamic key access: `obj?.[prop]`
  - Function call: `obj.method?.()`


6. OBJECT-ORIENTED JAVASCRIPT [Importance: 8.5/10]
--------------------------------------------------------------------------------
* Constructor Functions:
  Used to create objects with a shared blueprint.
  - Named with a Capital letter (PascalCase).
  - Executed with the `new` operator.
  ```javascript
  function User(name) {
    this.name = name;
    this.isAdmin = false;
  }
  let user = new User("Jack");
  ```

* What happens under the hood during a `new` call?
  1. A new empty object is created and assigned to `this`.
  2. The function body is executed (modifying `this` properties).
  3. The value of `this` is implicitly returned (unless an object is returned explicitly).

* Prototypes & Prototypal Inheritance:
  - Every JavaScript object has a private property `[[Prototype]]` linked to another object, 
    forming a Prototype Chain.
  - `__proto__` (getter/setter) points to `[[Prototype]]` of the object.
  - `prototype` is a property on **constructor functions** (not instances) that determines 
    what the `[[Prototype]]` link will be for instances created by that constructor.
  - Prototype lookup: If a property is not found on an object, JS looks up the chain 
    (`obj.__proto__` -> `obj.__proto__.__proto__`) until it either finds the property or 
    hits `null` (end of chain).
  ```javascript
  const animal = { eats: true };
  const rabbit = Object.create(animal); // sets rabbit's prototype to animal
  console.log(rabbit.eats); // true (inherited)
  console.log(rabbit.hasOwnProperty('eats')); // false (property is on prototype)
  ```

* ES6 Classes:
  A template wrapper around JavaScript's prototype-based inheritance.
  - Classes are hoisted (the name is registered) but enter the TDZ like `let`/`const` — 
    accessing a class before its declaration throws a `ReferenceError`.
  - Inherit using the `extends` keyword.
  - Subclass constructors must call `super()` before accessing `this`.
  - Class bodies implicitly run in strict mode.
  ```javascript
  class Animal {
    constructor(name) { this.name = name; }
  }

  class Dog extends Animal {
    constructor(name, breed) {
      super(name); // Required call
      this.breed = breed;
    }
  }
  ```


7. ASYNCHRONOUS JAVASCRIPT & PROMISES [Importance: 10/10]
--------------------------------------------------------------------------------
* Callbacks & Callback Hell:
  - Callback: A function passed into another function as an argument, which is executed at a later time (often after an asynchronous operation finishes).
  - Callback Hell (Pyramid of Doom): A situation where multiple nested callbacks are chained together, making the code extremely hard to read, maintain, and debug.
  - Example:
    ```javascript
    getData(function(a) {
      getMoreData(a, function(b) {
        getEvenMoreData(b, function(c) {
          console.log(c);
        });
      });
    });
    ```

* Promises:
  - Definition: An object representing the eventual completion (or failure) of an asynchronous operation and its resulting value.
  - Promise States:
    1. `pending`: Initial state, neither fulfilled nor rejected.
    2. `fulfilled`: Meaning that the operation was completed successfully.
    3. `rejected`: Meaning that the operation failed.
  - Promise Chaining: Using `.then()` to handle fulfillment, `.catch()` to handle rejection, and `.finally()` to run cleanup code regardless of the outcome. Each method returns a new Promise, allowing linear chains instead of nested callbacks.
    ```javascript
    fetchData()
      .then(result => processData(result))
      .then(processed => saveData(processed))
      .catch(error => console.error("Error occurred:", error))
      .finally(() => console.log("Operation finished."));
    ```

* async / await:
  - Syntactic sugar built on top of promises to write asynchronous code that reads like synchronous code.
  - `async` keyword: Placed before a function declaration to make it return a Promise.
  - `await` keyword: Can only be used inside an `async` function. It pauses code execution until the promise settles (resolves or rejects).
  - Error Handling: Managed cleanly using standard `try...catch` blocks.
  ```javascript
  async function performTask() {
    try {
      const data = await fetchData();
      const processed = await processData(data);
      console.log(processed);
    } catch (err) {
      console.error("Caught error:", err);
    }
  }
  ```

* Promise Concurrency Utilities:
  1. `Promise.all(iterable)`:
     - Fulfills when **all** promises in the input resolve; returns an array of results.
     - Rejects immediately if **any** input promise rejects (fail-fast behavior).
  2. `Promise.allSettled(iterable)`:
     - Waits for **all** input promises to settle (either fulfill or reject).
     - Returns an array of objects describing the outcome of each promise (e.g., `{ status: "fulfilled", value: val }` or `{ status: "rejected", reason: err }`). Never rejects.
  3. `Promise.any(iterable)`:
     - Fulfills as soon as **any** (the first) input promise fulfills.
     - Rejects with an `AggregateError` only if **all** input promises reject.
  4. `Promise.race(iterable)`:
     - Settles (fulfills or rejects) as soon as **one** of the input promises settles.

* Timers (setTimeout and setInterval):
  - `setTimeout(callback, delay, ...args)`:
    - Schedules a single execution of a callback function after the specified delay (in milliseconds).
    - Returns a numeric timer ID. Can be cancelled before execution using `clearTimeout(timerId)`.
  - `setInterval(callback, delay, ...args)`:
    - Repeatedly invokes a callback function at fixed intervals of the specified delay.
    - Returns a numeric interval ID. Can be stopped using `clearInterval(intervalId)`.

* The JavaScript Concurrency Model & Event Loop:
  - Call Stack:
    - A single-threaded, Last-In-First-Out (LIFO) stack that keeps track of the function execution context currently running.
  - Web APIs:
    - APIs provided by the browser environment (e.g., DOM methods, `fetch()`, `setTimeout`). They execute tasks outside the JavaScript engine's single thread. When a Web API completes its task, it pushes the callback to the appropriate queue.
  - Callback Queue (Task Queue):
    - A queue containing callbacks from standard asynchronous Web APIs like `setTimeout`, `setInterval`, or DOM event handlers.
  - Microtask Queue:
    - A higher-priority queue containing callbacks from Promise settlements (`.then`, `.catch`, `.finally`), `queueMicrotask`, or `MutationObserver`.
  - The Event Loop:
    - The coordination mechanism that continuously monitors the Call Stack and the queues.
    - **Step-by-step Execution Flow**:
      1. Check if the Call Stack is empty.
      2. If the Call Stack is empty, drain the **entire** Microtask Queue (including any microtasks added while draining).
      3. Render updates (if in browser and needed).
      4. Pull the first task from the Callback Queue and push it onto the Call Stack.
      5. Repeat the process.
    - Note: Because microtasks have absolute priority, they will always execute before the next timer callback.

* `this` in Context of Async Callbacks:
  - Inside standard callbacks, `this` can dynamically bind based on the caller or default 
    to the global scope (or `undefined` in strict mode).
  - Inside arrow functions, `this` is lexically bound to the scope where the arrow 
    function was originally defined, which is safer inside promise/timeout callbacks.


8. EVENT LOOP & BROWSER EVENTS [Importance: 10/10]
--------------------------------------------------------------------------------
* Event Loop:
  The core mechanism that enables single-threaded, non-blocking asynchronous execution.
  - When the Call Stack is empty, the Event Loop pulls tasks from the queues and pushes 
    them onto the Call Stack.

* Task Queue (Callback Queue) vs. Microtask Queue:
  1. Task Queue: Handles standard asynchronous callbacks (e.g. `setTimeout`, `setInterval`, 
     DOM event handlers).
  2. Microtask Queue: Handles high-priority asynchronous callbacks (e.g. `Promise.then`, 
     `catch`, `finally`, `queueMicrotask`, `MutationObserver`).
  
* Execution Priority:
  - Microtasks always have higher priority than standard tasks.
  - The Event Loop will drain the *entire* Microtask Queue before picking up the next 
    task from the Task Queue.
  - Consequently, promise resolution runs before `setTimeout` even if scheduled for the 
    same time.

* Event Propagation (Bubbling vs. Capturing):
  - Capturing Phase (Trickle-down): The event starts from the window and flows down 
    the DOM tree to the target element. Enabled by passing `true` as the third parameter 
    in `addEventListener('click', handler, true)`.
  - Target Phase: The event reaches the target element.
  - Bubbling Phase (Bubble-up): The event bubbles up from the target element back up to 
    the window. This is the default behavior.
  - Methods:
    - `event.stopPropagation()`: Stops the event from propagating further up or down the 
      DOM tree (prevents bubbling or capturing).
    - `event.stopImmediatePropagation()`: Stops propagation AND prevents any other 
      handlers on the same element from executing.
    - `event.preventDefault()`: Cancels the browser's default behavior (e.g., following 
      a link or submitting a form) but does not stop propagation.

* Event Delegation:
  A pattern of attaching a single event listener to a parent element instead of adding 
  individual listeners to all children. Uses event bubbling to catch events from children.
  - Great for performance (saves memory) and automatically works for dynamically added 
    child elements.
  ```javascript
  document.getElementById("parent-list").addEventListener("click", function(event) {
    if (event.target && event.target.nodeName === "LI") {
      console.log("List item clicked:", event.target.textContent);
    }
  });
  ```


9. DATA STRUCTURES & COLLECTIONS [Importance: 8/10]
--------------------------------------------------------------------------------
* Array Iteration & Transformation Methods:
  - map(callback): Creates a new array populated with the results of calling a provided function on every element in the calling array.
    ```javascript
    const numbers = [1, 2, 3];
    const doubled = numbers.map(x => x * 2); // [2, 4, 6]
    ```
  - filter(callback): Creates a shallow copy of a portion of a given array, filtered down to just the elements from the given array that pass the test implemented by the provided function.
    ```javascript
    const words = ['spray', 'limit', 'elite', 'exuberant'];
    const longWords = words.filter(word => word.length > 6); // ["exuberant"]
    ```
  - reduce(callback, initialValue): Executes a user-supplied "reducer" callback function on each element of the array, passing in the return value from the calculation on the preceding element. The final result of running the reducer across all elements of the array is a single value.
    ```javascript
    const arr = [1, 2, 3, 4];
    const sum = arr.reduce((accumulator, currentValue) => accumulator + currentValue, 0); // 10
    ```
  - forEach(callback): Executes a provided function once for each array element. Unlike `map`, it always returns `undefined` and does not chain.
    ```javascript
    const items = ['a', 'b', 'c'];
    items.forEach(item => console.log(item));
    ```

* Array Search & Condition Methods:
  - find(callback): Returns the value of the first element in the provided array that satisfies the provided testing function. If no values satisfy the testing function, `undefined` is returned.
    ```javascript
    const list = [5, 12, 8, 130, 44];
    const found = list.find(element => element > 10); // 12
    ```
  - findIndex(callback): Returns the index of the first element in the array that satisfies the provided testing function. Otherwise, it returns `-1`.
    ```javascript
    const index = list.findIndex(element => element > 10); // 1
    ```
  - some(callback): Tests whether at least one element in the array passes the test implemented by the provided function. Returns a boolean.
    ```javascript
    const hasNegative = [1, -2, 3].some(x => x < 0); // true
    ```
  - every(callback): Tests whether all elements in the array pass the test implemented by the provided function. Returns a boolean.
    ```javascript
    const allPositive = [1, 2, 3].every(x => x > 0); // true
    ```

* Array Sorting, Flattening & Restructuring:
  - sort(compareFunction): Sorts the elements of an array in place and returns the reference to the same array. By default, it converts elements to strings and sorts them according to UTF-16 code unit values (which can yield unexpected results for numbers: e.g., `[10, 2].sort()` returns `[10, 2]`).
    ```javascript
    const nums = [40, 100, 1, 5, 25, 10];
    nums.sort((a, b) => a - b); // Ascending sort: [1, 5, 10, 25, 40, 100]
    ```
  - flat(depth): Creates a new array with all sub-array elements concatenated into it recursively up to the specified depth. Defaults to `1`.
    ```javascript
    const nested = [1, 2, [3, 4, [5, 6]]];
    console.log(nested.flat());    // [1, 2, 3, 4, [5, 6]]
    console.log(nested.flat(2));   // [1, 2, 3, 4, 5, 6]
    console.log(nested.flat(Infinity)); // Flattens all nested arrays
    ```
  - flatMap(callback): Returns a new array formed by applying a given callback function to each element of the array, and then flattening the result by one level. It is equivalent to a `map()` followed by a `flat(1)`, but slightly more efficient.
    ```javascript
    const sentence = ["hello world", "good morning"];
    const wordsList = sentence.flatMap(x => x.split(" ")); // ["hello", "world", "good", "morning"]
    ```

* Essential Array Interview Questions & Key Differences:
  - map() vs. forEach():
    - `map()` returns a new array and does not mutate the original array (if pure). It is chainable (e.g., `.map().filter()`).
    - `forEach()` does not return anything (`undefined`), executes side effects, and is not chainable.
  - filter() vs. find():
    - `filter()` returns a new array containing all elements that match the condition (can be empty).
    - `find()` returns only the first matching element value (or `undefined` if no match is found).
  - slice() vs. splice():
    - `slice(start, end)`: Returns a shallow copy of a portion of an array into a new array object. It **does not mutate** the original array.
    - `splice(start, deleteCount, ...items)`: Changes the contents of an array by removing or replacing existing elements and/or adding new elements in place. It **mutates** the original array.
  - push() vs. concat():
    - `push(...items)`: Adds one or more elements to the end of an array and returns the new length. It **mutates** the original array in place.
    - `concat(...items)`: Merges two or more arrays/values. It returns a brand new array and **does not mutate** the original arrays.

* Object and Array Destructuring:
  Extracted assignments from arrays and objects:
  ```javascript
  // Object:
  const classDetails = { strength: 78, benches: 39 };
  const { strength: classStrength, benches: classBenches } = classDetails;
  
  // Array:
  const arr = [1, 2, 3];
  const [first, second] = arr;
  ```

* Rest Parameter (`...` in declarations) vs. Spread Operator (`...` in execution):
  - Rest Parameter: Gathers parameters into an array. Must be the last parameter.
    ```javascript
    function sumAll(...args) { return args.reduce((a, b) => a + b); }
    ```
  - Spread Operator: Spreads array elements or object properties.
    ```javascript
    let arr2 = [...arr1, 4, 5];
    ```

* WeakSet & WeakMap:
  - `WeakSet`: Stores unique objects. Objects are held weakly, meaning they can be 
    garbage-collected if there are no other references to them.
  - `WeakMap`: Stores key/value pairs where keys must be objects (or unique symbols) 
    and values can be any type. Keys are held weakly, preventing memory leaks if the key 
    objects are deleted elsewhere. Non-enumerable.

* charAt(index):
  Returns the character at the specified index in a string.

* RegExp Methods: `exec()` vs. `test()`:
  - `exec()`: Searches a string for a match and returns an array containing matched 
    groups, or `null` if no match is found.
    ```javascript
    let match = /x/.exec("xyz"); // Returns array with match details
    ```

11. ADVANCED OPTIMIZATION TECHNIQUES [Importance: 9/10]
--------------------------------------------------------------------------------
* Memoization:
  An optimization technique that caches function computation results.
  ```javascript
  // Fibonacci Memoization Example:
  function memoizedFib() {
    const cache = {};
    return function fib(n) {
      if (n in cache) return cache[n];
      if (n <= 1) return n;
      cache[n] = fib(n - 1) + fib(n - 2);
      return cache[n];
    };
  }
  const fib = memoizedFib();
  console.log(fib(40)); // Much faster than standard recursion
  ```

* Generators:
  Functions that can pause and resume execution using `function*` and `yield`.
  - Pause execution and return value on the fly without storing all results in memory.
  - Yields object format: `{ value: X, done: boolean }`.
  - Can safely support infinite loop generators without freezing execution.
  ```javascript
  function* idGenerator() {
    let id = 1;
    while (true) {
      yield id++;
    }
  }
  const gen = idGenerator();
  console.log(gen.next().value); // 1
  console.log(gen.next().value); // 2
  console.log(gen.next().value); // 3
  ```
  - Comparison: Async/Await vs. Generators:
    - Generators execute yield-by-yield manually when `.next()` is called, yielding `{ value: X, done: boolean }` objects.
    - Async/Await functions execute sequentially to completion automatically, wrapping the final returned value in a Promise (or throwing an error).
    - Async/Await is a specialized implementation designed specifically for cleaner sequential asynchronous control flow.

* Debouncing & Throttling:
  - Debouncing: Runs the function only after a certain delay has elapsed since the 
    last invocation (e.g. typing search queries).
    ```javascript
    function debounce(fn, delay) {
      let timerId = null;
      return function (...args) {
        if (timerId) clearTimeout(timerId);
        timerId = setTimeout(() => {
          fn.apply(this, args);
        }, delay);
      };
    }
    ```
  - Throttling: Limits execution to at most once every X milliseconds (e.g. scroll handlers).
    ```javascript
    function throttle(fn, delay) {
      let lastExecutionTime = 0;
      return function (...args) {
        const now = Date.now();
        if (now - lastExecutionTime >= delay) {
          lastExecutionTime = now;
          fn.apply(this, args);
        }
      };
    }
    ```


12. MAP vs. PLAIN OBJECT
--------------------------------------------------------------------------------
* `Map`:
  A keyed collection where any value (including objects) can be a key.
  - Maintains insertion order.
  - Has a built-in `.size` property.
  - Never has key collisions from prototype properties.
  - Iterable directly with `for..of`.
  ```javascript
  const map = new Map();
  map.set('name', 'Alice');
  map.set(42, 'answer');
  map.set({}, 'obj-key');
  console.log(map.size); // 3
  ```

* When to use `Map` over `Object`:
  - Keys are not strings/symbols (e.g., DOM nodes, objects).
  - Key insertion order must be guaranteed.
  - Frequent add/delete operations (Map is optimized for this).
  - Use plain `Object` when keys are strings and you need JSON serialization.


13. WEAKREF [Importance: 5/10]
--------------------------------------------------------------------------------
* `WeakRef`:
  Holds a weak reference to an object without preventing it from being garbage-collected.
  - Use `.deref()` to access the target; returns `undefined` if it has been GC'd.
  ```javascript
  let obj = { name: 'target' };
  const ref = new WeakRef(obj);
  obj = null; // Original reference dropped

  const target = ref.deref();
  if (target) {
    console.log(target.name); // 'target' (if not yet GC'd)
  }
  ```
  - Pairs naturally with `WeakMap`/`WeakSet` for cache/registry patterns.
  - Do not rely on `WeakRef` for critical program logic — GC timing is non-deterministic.


14. PROXY & REFLECT [Importance: 7/10]
--------------------------------------------------------------------------------
* Proxy:
  Wraps an object and intercepts fundamental operations (get, set, delete, etc.) via traps.
  ```javascript
  const handler = {
    get(target, prop) {
      return prop in target ? target[prop] : `Property '${prop}' not found`;
    },
    set(target, prop, value) {
      if (typeof value !== 'number') throw new TypeError('Only numbers allowed');
      target[prop] = value;
      return true;
    }
  };

  const proxy = new Proxy({}, handler);
  proxy.age = 25;
  console.log(proxy.age);    // 25
  console.log(proxy.name);   // "Property 'name' not found"
  ```

* Reflect:
  A built-in object providing methods matching the Proxy trap signatures. Used to 
  forward operations to the original target inside traps.
  ```javascript
  get(target, prop, receiver) {
    return Reflect.get(target, prop, receiver); // Correct forwarding
  }
  ```
  - Key insight: `Reflect` methods mirror Proxy traps 1-to-1, making them natural partners.


15. MODERN ARRAY UTILITIES [Importance: 8.5/10]
--------------------------------------------------------------------------------
* `Array.at(index)`:
  Accepts positive and negative integers. Negative integers count back from the end.
  ```javascript
  const arr = [1, 2, 3, 4, 5];
  console.log(arr.at(0));   // 1
  console.log(arr.at(-1));  // 5 (last element)
  console.log(arr.at(-2));  // 4
  ```
  Advantage over `arr[arr.length - 1]`: cleaner and works on strings/TypedArrays too.

* `Array.from()`:
  Creates an array from an iterable or array-like object.
  ```javascript
  Array.from('hello');           // ['h', 'e', 'l', 'l', 'o']
  Array.from({length: 3}, (_, i) => i); // [0, 1, 2]
  ```

* `Array.findIndex()` vs `Array.indexOf()`:
  - `indexOf(val)`: finds by strict equality.
  - `findIndex(fn)`: finds by predicate — works for objects and complex conditions.
  ```javascript
  const users = [{ id: 1 }, { id: 2 }];
  users.findIndex(u => u.id === 2); // 1
  ```


16. ES MODULES (ESM) [Importance: 8/10]
--------------------------------------------------------------------------------
* Named vs. Default Exports:
  ```javascript
  // Named exports — multiple per file:
  export const PI = 3.14;
  export function add(a, b) { return a + b; }

  // Default export — one per file:
  export default class MyClass { ... }
  ```

* Importing:
  ```javascript
  import { PI, add } from './math.js';       // Named
  import MyClass from './MyClass.js';          // Default
  import * as math from './math.js';           // Namespace import
  import { add as sum } from './math.js';      // Aliased
  ```

* Dynamic Import (lazy loading):
  Returns a Promise — useful for code splitting.
  ```javascript
  const { add } = await import('./math.js');
  ```

* ESM vs. CommonJS (CJS):
  | Feature            | ESM (`import/export`)   | CJS (`require/module.exports`) |
  |--------------------|-------------------------|--------------------------------|
  | Environment        | Browser + Node.js       | Node.js (legacy)               |
  | Loading            | Static (compile-time)   | Dynamic (runtime)              |
  | `this` at top level| `undefined`             | `module.exports`               |
  | Tree-shakeable     | Yes                     | No                             |


17. ERROR HANDLING PATTERNS [Importance: 7.5/10]
--------------------------------------------------------------------------------
* Runtime Error Types:
  - `ReferenceError`: Accessed a non-existent variable.
  - `TypeError`: Value is not the expected type.
  - `SyntaxError`: Code structure is invalid.
  - `RangeError`: Number is outside the allowed range.

* try / catch / finally:
  ```javascript
  try {
    riskyOperation();
  } catch (err) {
    console.error(err.message);
  } finally {
    cleanup(); // Always runs, even if an error was thrown
  }
  ```

* Custom Error Classes:
  Extend the built-in `Error` class to create domain-specific error types.
  ```javascript
  class ValidationError extends Error {
    constructor(message, field) {
      super(message);
      this.name = 'ValidationError';
      this.field = field;
    }
  }

  try {
    throw new ValidationError('Required field missing', 'email');
  } catch (err) {
    if (err instanceof ValidationError) {
      console.log(`${err.name} on '${err.field}': ${err.message}`);
    }
  }
  ```

* Async Error Handling:
  ```javascript
  // Option 1: try/catch with async/await
  async function fetchUser(id) {
    try {
      const res = await fetch(`/api/users/${id}`);
      if (!res.ok) throw new Error(`HTTP error: ${res.status}`);
      return await res.json();
    } catch (err) {
      console.error('Fetch failed:', err);
    }
  }

  // Option 2: .catch() on the promise chain
  fetch('/api/data')
    .then(res => res.json())
    .catch(err => console.error(err));
  ```

* Unhandled Promise Rejections:
  Always handle rejections. In Node.js/browsers, unhandled rejections can crash the 
  process or print warnings:
  ```javascript
  process.on('unhandledRejection', (reason) => {
    console.error('Unhandled rejection:', reason);
  });
  ```

18. MODERN ES6+ FEATURES SUMMARY [Importance: 9/10]
--------------------------------------------------------------------------------
* Template Literals:
  - Enclosed by backticks (`` ` ``) instead of single/double quotes.
  - Supports multi-line strings directly, and string interpolation using `${expression}`.
  - Example:
    ```javascript
    const name = "Sahil";
    const msg = `Hello, ${name}!
    Welcome to the team.`;
    ```

* Spread Operator vs. Rest Operator:
  - Both use the `...` syntax, but serve opposite purposes:
  - Spread Operator (Unpacks elements):
    - Expands arrays or objects into individual elements or properties.
    - Used in function arguments, array literals, or object literals.
    - Example: `const newArr = [...oldArr, 4, 5];`
  - Rest Operator (Packs elements):
    - Gathers multiple remaining elements/arguments into a single array.
    - Used in function parameter declarations or destructuring assignments.
    - Example:
      ```javascript
      function sum(...numbers) { // Rest gathers arguments
        return numbers.reduce((a, b) => a + b);
      }
      ```

* Destructuring Assignment:
  - Allows unpacking values from arrays or properties from objects into distinct variables.
  - Example:
    ```javascript
    // Object destructuring with renaming and defaults:
    const { name, role: userRole, age = 18 } = { name: "Alice", role: "Admin" };
    // Array destructuring:
    const [first, second, ...rest] = [1, 2, 3, 4, 5];
    ```

* Default Parameters:
  - Allows parameters to be initialized with default values if no value or `undefined` is passed.
  - Example:
    ```javascript
    function multiply(a, b = 1) {
      return a * b;
    }
    multiply(5); // Returns 5 (b defaults to 1)
    ```

* Optional Chaining (`?.`):
  - Safely reads nested properties or calls methods without throwing an error if a reference is nullish (`null` or `undefined`). Returns `undefined` instead.
  - Example:
    ```javascript
    const user = { profile: { getName: () => "Bob" } };
    console.log(user?.profile?.getName?.()); // "Bob"
    console.log(user?.address?.city); // undefined (does not throw)
    ```

* Nullish Coalescing Operator (`??`):
  - A logical operator that returns its right-hand side operand when its left-hand side operand is `null` or `undefined`.
  - Contrast with logical OR (`||`): `||` returns the right-hand operand for *any* falsy value (e.g., `""`, `0`, `false`), which is often a bug when those falsy values are valid inputs.
  - Example:
    ```javascript
    const count = 0;
    console.log(count ?? 10); // 0 (since 0 is not null/undefined)
    console.log(count || 10); // 10 (since 0 is falsy)
    ```

* Modules (Import / Export):
  - Allows splitting code into reusable files using ES Modules syntax:
  - Named Exports/Imports:
    ```javascript
    // file: utils.js
    export const log = (msg) => console.log(msg);
    // file: main.js
    import { log } from './utils.js';
    ```
  - Default Exports/Imports:
    ```javascript
    // file: api.js
    export default function fetchApi() {}
    // file: main.js
    import fetchApi from './api.js';
    ```
