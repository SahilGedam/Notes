1. JAVASCRIPT BASICS & EXECUTION CONTEXT [Importance: 9/10]
--------------------------------------------------------------------------------
* What is JavaScript?
  JavaScript is a lightweight, cross-platform, single-threaded, and just-in-time 
  (JIT) compiled programming language. It is traditionally known as the scripting 
  language for webpages, but it is now widely used in many non-browser environments 
  (e.g., Node.js).
  - It is weakly (dynamically) typed.
  - It supports both imperative and declarative paradigms.
  - Contains a standard library of built-in objects (Array, Date, Math) and core 
    language elements (operators, control structures, statements).

* Strict Mode:
  Introduced in ECMAScript 5, strict mode is a way to opt into a restricted variant 
  of JavaScript that throws errors for silent failure cases.
  - Enabled by adding the `'use strict'` directive at the top of a script or function.
  - Characteristics:
    - Prevents duplicate parameter names (e.g. `function x(a, a) {}` is a SyntaxError).
    - Restricts naming variables/parameters after reserved words (like `eval`, `arguments`).
    - Prevents implicit creation of global variables (assigning to an undeclared 
      variable throws a `ReferenceError`).
    - In strict mode, `this` in global function calls evaluates to `undefined` 
      instead of the `window`/global object.

* Transpilers & Polyfills:
  Tools used to ensure modern JavaScript features run on older browser engines.
  - Transpilers (e.g. Babel): Source-to-source compilers that translate new syntax 
    into equivalent older syntax.
    - Example: Transpiling the nullish coalescing operator:
      ```javascript
      // Modern input:
      height = height ?? 100;
      
      // Transpiled output:
      height = (height !== undefined && height !== null) ? height : 100;
      ```
  - Polyfills (e.g. core-js): Scripts that add missing built-in functions/methods.
    - Example: Fills in `Math.trunc(1.23)` for engines lacking it.

* Global Object:
  Provides variables and functions available anywhere in the scope.
  - Browser: `window`
  - Node.js: `global`
  - Standardized (across all environments): `globalThis`

* Eval Function:
  Allows execution of code represented as a string:
  ```javascript
  let result = eval('2 + 2'); // Returns 4
  ```
  Note: Avoid using `eval` in production as it poses severe security risks and 
  degrades execution performance.

* Types of Errors in JavaScript:
  - Syntax Error: Mistakes or spelling/structure problems in the code that cause the 
    program to fail parsing and not execute at all (or stop running).
  - Runtime Error: Valid syntax, but execution fails at that point (e.g., accessing a 
    property on `null`, calling a non-function). The program starts running but throws 
    an exception mid-execution.
  - Logical Error: Program logic is incorrect, but the syntax is valid. The application 
    runs without throwing browser errors, but produces incorrect results. These are often 
    harder to debug.


2. DATA TYPES & COERCION [Importance: 8.5/10]
--------------------------------------------------------------------------------
* Categories of Data Types:
  1. Primitive Types: Stored directly in the execution stack. They are immutable and 
     copied by value.
     - `String`, `Number`, `Boolean`, `BigInt`, `Undefined`, `Null`, `Symbol`
  2. Non-Primitive Types (Objects): Stored in the memory heap. They are mutable and 
     references to them are copied by value (sharing).
     - `Object` (includes Arrays, Functions, Dates, etc.)

* `typeof` Operator Output Examples:
  ```javascript
  typeof "John Doe" // "string"
  typeof 3.14       // "number"
  typeof true       // "boolean"
  typeof 42n        // "bigint"
  typeof undefined  // "undefined"
  typeof null       // "object" (acknowledged legacy bug in JS)
  typeof Symbol()   // "symbol"
  typeof {}         // "object"
  typeof function(){} // "function"
  ```

* Implicit Type Coercion:
  Automatic conversion of values from one data type to another during operations.
  - String Coercion: Triggered by the binary `+` operator when one operand is a string.
    ```javascript
    "3" + 4 // "34" (number 4 is coerced to "4")
    ```
  - Boolean Coercion: Occurs in logical checks (loops, `if` statements).
    - Falsy values: `false`, `0`, `-0`, `0n`, `""`, `null`, `undefined`, and `NaN`.
    - Everything else is truthy.
  - Equality Coercion: Occurs during loose equality comparisons (`==`).

* Loose (`==`) vs. Strict (`===`) Equality:
  - `==` (Loose): Compares values after performing type coercion.
    ```javascript
    "5" == 5 // true
    ```
  - `===` (Strict): Compares both values and types without coercion.
    ```javascript
    "5" === 5 // false
    ```
  - Classic Interview Question: Why does `[] == ![]` evaluate to `true`?
    1. `!` has higher precedence than `==`. So `![]` is evaluated first. Since arrays are truthy, `![]` becomes `false`.
    2. The expression is now `[] == false`.
    3. Loose equality (`==`) with a boolean coerces both to numbers: `false` becomes `0`.
    4. The expression is now `[] == 0`.
    5. The array `[]` is coerced to a primitive. `[].toString()` is `""` (empty string).
    6. The expression is now `"" == 0`.
    7. Empty string `""` coerced to a number is `0`.
    8. The expression becomes `0 == 0`, which is `true`.

* NaN Property (Not-a-Number):
  Represents an invalid or unrepresentable numerical calculation.
  - `typeof NaN` returns `"number"`.
  - Use global `isNaN()` or `Number.isNaN()` to check. Note that `isNaN()` coerces 
    its input to a number before checking, while `Number.isNaN()` does not.
    ```javascript
    isNaN("hello")        // true (coerces "hello" to NaN)
    Number.isNaN("hello") // false (type is string, not NaN number)
    ```

* Infinity:
  A numeric property representing positive infinity (greater than any number).
  `typeof Infinity` returns `"number"`.

* Symbols:
  A primitive type representing unique, immutable identifiers.
  ```javascript
  let id1 = Symbol("id");
  let id2 = Symbol("id");
  console.log(id1 === id2); // false (always unique)
  ```
  - Auto-conversion: Symbols do not auto-convert to string. Direct string output 
    throws a TypeError; call `.toString()` explicitly.
  - Object Properties: Symbols are skipped by `for..in` loops and `Object.keys()`, 
    but are copied by `Object.assign()`. To retrieve symbol-keyed properties, use 
    `Object.getOwnPropertySymbols(obj)`.
  - Global Registry: Use `Symbol.for(key)` to retrieve or create a global symbol. 
    `Symbol.keyFor(sym)` returns its key name.

* Object to Primitive Conversion (Hints):
  JavaScript uses three internal "hints" when converting objects to primitives:
  - `"string"`: Used when an operation expects a string (e.g. `alert(obj)`).
  - `"number"`: Used when math is performed (e.g. `+obj`, `obj1 - obj2`).
  - `"default"`: Used when the operator is unsure (e.g. binary `+` or loose equality `==`).


3. VARIABLES & SCOPING [Importance: 9/10]
--------------------------------------------------------------------------------
* Scope Types:
  1. Global Scope: Variables declared outside any function or block, accessible 
     anywhere in the code.
  2. Local / Function Scope: Variables declared inside a function, accessible only 
     within that function.
  3. Block Scope: Variables declared using `let` and `const` inside `{}` blocks, 
     inaccessible outside that block. Note: `var` is NOT block-scoped.

* var vs. let vs. const:
  - `var`: Function-scoped. Can be redeclared. Hoisted and initialized to `undefined`.
  - `let`: Block-scoped. Cannot be redeclared. Hoisted but not initialized (TDZ).
  - `const`: Same scoping and hoisting properties as `let`, but cannot be reassigned 
    and must be initialized during declaration.
  - Single-Statement Context Restriction:
    `let` begins declarations, not statements. You cannot use a lone `let` declaration 
    as the body of a block:
    ```javascript
    if (true) let a = 1; // SyntaxError: Lexical declaration cannot appear in a single-statement context
    ```

* Hoisting:
  The process whereby the JavaScript interpreter processes declarations of functions, 
  variables, classes, and imports before executing the code.
  - Declarations are hoisted, not initializations.
  - Function declarations are fully hoisted (can be called before defined).
  - `var` declarations are hoisted and initialized to `undefined`.
  - `let` and `const` declarations are hoisted but remain uninitialized.

* Temporal Dead Zone (TDZ):
  The period from the start of the block until the execution reaches the declaration line. 
  Accessing a variable in the TDZ throws a `ReferenceError`.
  ```javascript
  {
    // TDZ for variable 'a' starts here
    console.log(a); // ReferenceError
    let a = 10;     // TDZ ends
  }
  ```

* Scope Chain:
  If a variable is not found in the local scope, the JS engine searches the outer 
  lexical scopes sequentially until it reaches the global scope. If it is still not 
  found, a `ReferenceError` is thrown.


4. FUNCTIONS & CLOSURES [Importance: 10/10]
--------------------------------------------------------------------------------
* Function Declarations vs. Expressions:
  - Function Declaration: Declared as a standalone statement. Hoisted.
    ```javascript
    function greet() { return "Hello"; }
    ```
  - Function Expression: Created inside an expression. Not hoisted.
    ```javascript
    const greet = function() { return "Hello"; };
    ```
  - Scope in Block: In strict mode, function declarations in blocks are block-scoped.

* Arrow Functions:
  A concise syntax introduced in ES6.
  ```javascript
  const arrowAdd = (a, b) => a + b;
  ```
  - Multiline: Braces `{}` require an explicit `return` statement.
  - Lexical `this`: Arrow functions do not bind their own `this`. They inherit `this` 
    from the enclosing lexical environment in which they are defined.

* Immediately Invoked Function Expressions (IIFE):
  Functions that run immediately after definition (pronounced "Iffy").
  ```javascript
  (function() {
    // Executed immediately
  })();
  ```

* new Function Syntax:
  Allows dynamically creating a function from strings:
  ```javascript
  let sum = new Function('a', 'b', 'return a + b');
  console.log(sum(1, 2)); // 3
  ```

* Callbacks:
  Passing a function as an argument to another function.
  ```javascript
  function multiplyBy2(sum) {
    console.log(sum * 2);
  }

  function operationOnSum(num1, num2, callback) {
    let sum = num1 + num2;
    callback(sum);
  }

  operationOnSum(5, 5, multiplyBy2); // Outputs 20
  ```

* Higher-Order Functions:
  Functions that take other functions as arguments, return them, or both (due to 
  functions being first-class citizens).
  ```javascript
  function higherOrder2() {
    return function() { return "Result"; };
  }
  let x = higherOrder2();
  x(); // Returns "Result"
  ```

* Closures:
  A closure is the combination of a function bundled together with references to 
  its surrounding state (its lexical environment). This gives an inner function 
  access to an outer function's variables even after the outer function has finished 
  executing.
  ```javascript
  function randomFunc() {
    var obj1 = { name: "Vivian", age: 45 };
    return function() {
      console.log(obj1.name + " is awesome"); // Accesses obj1 from lexical env
    };
  }

  var initialiseClosure = randomFunc();
  initialiseClosure(); // Outputs "Vivian is awesome"
  ```

* Currying:
  Transforming a function of `N` arguments into `N` sequential functions of single arguments.
  ```javascript
  // Target Call Signature: add(1)(2)(3)
  
  // Curried Implementation:
  function add(a) {
    return function(b) {
      return function(c) {
        return a + b + c;
      };
    };
  }

  console.log(add(1)(2)(3)); // 6
  ```

* "this" Keyword & call(), apply(), bind():
  - `this` Keyword: Refers to the object that the function is a property of (the invoking object).
    - If called globally (non-strict mode): `this` refers to `window` / `globalThis`.
    - If called globally (strict mode): `this` is `undefined`.
    - Arrow functions: Inherit `this` lexically from their enclosing parent context (they do not have their own `this`).
    - Normal functions: `this` dynamically refers to the object invoking the function.
  - call(), apply(), bind() are methods used to explicitly set the `this` context:
    - `call()`: Invokes the function immediately with a given `this` value and arguments passed one by one.
    - `apply()`: Invokes the function immediately with a given `this` value and arguments passed as a single array.
    - `bind()`: Returns a new function with `this` bound to the specified object, to be executed later.
  
  - Interview Code Example:
    ```javascript
    let person1 = {
      name: 'Sahil'
    };
    
    function printDetails(age, food_choice) {
      console.log(this.name, "is", age, "years old and is", food_choice);
    }
    
    printDetails.call(person1, 28, 'vegetarian'); // call (arguments individually)
    printDetails.apply(person1, [28, 'vegetarian']); // apply (arguments as array)
    
    let func = printDetails.bind(person1, 28, 'Vegetarian'); // bind (returns new function)
    func();
    ```
  - Function Borrowing Example:
    ```javascript
    var person = {
      age: 23,
      getAge: function() { return this.age; }
    };
    var person2 = { age: 54 };
    person.getAge.call(person2); // Returns 54 (person2 borrows getAge method)
    ```


5. OBJECTS & MEMORY MODEL [Importance: 9/10]
--------------------------------------------------------------------------------
* Object Basics:
  ```javascript
  let user = {}; // Object literal syntax
  user.name = "John"; // Dot access
  user["likes birds"] = true; // Bracket notation (required for multiword keys)
  delete user.name; // Property removal
  ```

* Object Property Loop & Checks:
  - Checking existence: `"key" in obj`
  - Iterating: `for (let key in obj) { ... }`

* Memory Model (Pass by Value / Sharing):
  JavaScript is strictly **pass-by-value**.
  - Primitives: The value is copied.
  - Objects: The reference (memory address) to the object is copied and passed by value.
    - Property mutations inside a function affect the outer object.
    - Reassigning the object parameter does not change the outer variable.
  ```javascript
  function modify(obj) {
    obj.prop = "mutated"; // Affects outer object
    obj = { newProp: 2 }; // Does not affect outer object (reassigned local copy of reference)
  }
  ```

* Object Equality Comparison:
  Two objects are equal only if they reference the same object in memory.
  ```javascript
  let a = {};
  let b = a;
  console.log(a === b); // true (same reference)
  
  let c = {};
  let d = {};
  console.log(c === d); // false (different references in heap)
  ```

* Restricting Object Mutations:
  - `Object.preventExtensions(obj)`: Prevents adding new properties.
  - `Object.seal(obj)`: Prevents adding/removing properties; sets `configurable: false`.
    Note: Existing property *values* can still be changed (unless also `writable: false`).
  - `Object.freeze(obj)`: Prevents adding/removing/updating properties; sets 
    `configurable: false` and `writable: false`. Note: This is a shallow freeze — 
    nested objects remain mutable.

* Shallow vs. Deep Copy:
  - Shallow Copy: Copies only the first level of keys. Nested objects are still shared 
    by reference.
    - Example: `Object.assign({}, original)` or using spread `const clone = { ...original }`.
  - Deep Copy: Copies all nested levels recursively.
    - Legacy Method: `JSON.parse(JSON.stringify(obj))` (fails with functions, Symbols, 
      RegExp, and loses `undefined` fields / Date objects are converted to strings).
    - Modern Native API: `structuredClone(obj)` (fully deep clones structures, supports 
      Dates, Sets, Maps, Arrays, etc. natively).

* Object Property Descriptors:
  You can control properties with fine-grained configuration using `Object.defineProperty(obj, prop, descriptor)`:
  - `writable`: If `true`, the value of the property can be changed.
  - `enumerable`: If `true`, the property shows up in `for..in` loops and `Object.keys()`.
  - `configurable`: If `true`, the property can be deleted and its descriptors modified.
  ```javascript
  let user = {};
  Object.defineProperty(user, "id", {
    value: 101,
    writable: false, // Cannot be modified
    enumerable: false, // Hidden in loops
    configurable: false // Cannot be deleted
  });
  ```

* Getters & Setters:
  Accessor properties inside object literals:
  ```javascript
  let obj = {
    get propName() { return this._val; },
    set propName(value) { this._val = value; }
  };
  ```

* Optional Chaining (`?.`):
  Stops evaluation and returns `undefined` if the operand before `?.` is `null` or 
  `undefined`.
  - Property access: `obj?.prop`
  - Dynamic key access: `obj?.[prop]`
  - Function call: `obj.method?.()`


6. OBJECT-ORIENTED JAVASCRIPT [Importance: 8.5/10]
--------------------------------------------------------------------------------
* Constructor Functions:
  Used to create objects with a shared blueprint.
  - Named with a Capital letter (PascalCase).
  - Executed with the `new` operator.
  ```javascript
  function User(name) {
    this.name = name;
    this.isAdmin = false;
  }
  let user = new User("Jack");
  ```

* What happens under the hood during a `new` call?
  1. A new empty object is created and assigned to `this`.
  2. The function body is executed (modifying `this` properties).
  3. The value of `this` is implicitly returned (unless an object is returned explicitly).

* Prototypes & Prototypal Inheritance:
  - Every JavaScript object has a private property `[[Prototype]]` linked to another object, 
    forming a Prototype Chain.
  - `__proto__` (getter/setter) points to `[[Prototype]]` of the object.
  - `prototype` is a property on **constructor functions** (not instances) that determines 
    what the `[[Prototype]]` link will be for instances created by that constructor.
  - Prototype lookup: If a property is not found on an object, JS looks up the chain 
    (`obj.__proto__` -> `obj.__proto__.__proto__`) until it either finds the property or 
    hits `null` (end of chain).
  ```javascript
  const animal = { eats: true };
  const rabbit = Object.create(animal); // sets rabbit's prototype to animal
  console.log(rabbit.eats); // true (inherited)
  console.log(rabbit.hasOwnProperty('eats')); // false (property is on prototype)
  ```

* ES6 Classes:
  A template wrapper around JavaScript's prototype-based inheritance.
  - Classes are hoisted (the name is registered) but enter the TDZ like `let`/`const` — 
    accessing a class before its declaration throws a `ReferenceError`.
  - Inherit using the `extends` keyword.
  - Subclass constructors must call `super()` before accessing `this`.
  - Class bodies implicitly run in strict mode.
  ```javascript
  class Animal {
    constructor(name) { this.name = name; }
  }

  class Dog extends Animal {
    constructor(name, breed) {
      super(name); // Required call
      this.breed = breed;
    }
  }
  ```


7. ASYNCHRONOUS JAVASCRIPT & PROMISES [Importance: 10/10]
--------------------------------------------------------------------------------
* Promises:
  An object representing the eventual completion or failure of an asynchronous operation.
  - States:
    - `pending`: Initial state.
    - `fulfilled`: Completed successfully.
    - `rejected`: Operation failed.
  - Promise objects are immutable (once settled, their state can never change).

* Chaining:
  Methods like `.then()`, `.catch()`, and `.finally()` associate actions with settled 
  promises and return new promises, allowing chaining:
  ```javascript
  doTask1()
    .then(res => doTask2(res))
    .catch(err => handleError(err))
    .finally(() => cleanup());
  ```

* Promise Concurrency Utilities:
  1. `Promise.all([p1, p2])`: Fulfills when all fulfill; rejects if *any* reject.
  2. `Promise.allSettled([p1, p2])`: Fulfills when all settle (fulfilled or rejected).
  3. `Promise.any([p1, p2])`: Fulfills when *any* fulfills; rejects with `AggregateError` 
     if all reject.
  4. `Promise.race([p1, p2])`: Settles when the first promise settles (resolves or 
     rejects).

* Modern Promise helpers:
  - `Promise.resolve(val)`: Creates a resolved promise.
  - `Promise.reject(reason)`: Creates a rejected promise.
  - `Promise.withResolvers()` (ES2024): Returns an object containing a new promise 
    along with its resolve and reject functions:
    ```javascript
    const { promise, resolve, reject } = Promise.withResolvers();
    setTimeout(() => resolve("Success"), 1000);
    ```

* async / await:
  Syntactic sugar built on top of promises to write async code in a synchronous style.
  `await` pauses execution of the async function until the promise settles.
  ```javascript
  async function fetchData() {
    try {
      let data = await apiCall();
      console.log(data);
    } catch (err) {
      console.error(err);
    }
  }
  ```

* `this` in Context of Async Callbacks:
  - Inside standard callbacks, `this` can dynamically bind based on the caller or default 
    to the global scope (or `undefined` in strict mode).
  - Inside arrow functions, `this` is lexically bound to the scope where the arrow 
    function was originally defined, which is safer inside promise/timeout callbacks.


8. EVENT LOOP & BROWSER EVENTS [Importance: 10/10]
--------------------------------------------------------------------------------
* Event Loop:
  The core mechanism that enables single-threaded, non-blocking asynchronous execution.
  - When the Call Stack is empty, the Event Loop pulls tasks from the queues and pushes 
    them onto the Call Stack.

* Task Queue (Callback Queue) vs. Microtask Queue:
  1. Task Queue: Handles standard asynchronous callbacks (e.g. `setTimeout`, `setInterval`, 
     DOM event handlers).
  2. Microtask Queue: Handles high-priority asynchronous callbacks (e.g. `Promise.then`, 
     `catch`, `finally`, `queueMicrotask`, `MutationObserver`).
  
* Execution Priority:
  - Microtasks always have higher priority than standard tasks.
  - The Event Loop will drain the *entire* Microtask Queue before picking up the next 
    task from the Task Queue.
  - Consequently, promise resolution runs before `setTimeout` even if scheduled for the 
    same time.

* Event Propagation (Bubbling vs. Capturing):
  - Capturing Phase (Trickle-down): The event starts from the window and flows down 
    the DOM tree to the target element. Enabled by passing `true` as the third parameter 
    in `addEventListener('click', handler, true)`.
  - Target Phase: The event reaches the target element.
  - Bubbling Phase (Bubble-up): The event bubbles up from the target element back up to 
    the window. This is the default behavior.
  - Methods:
    - `event.stopPropagation()`: Stops the event from propagating further up or down the 
      DOM tree (prevents bubbling or capturing).
    - `event.stopImmediatePropagation()`: Stops propagation AND prevents any other 
      handlers on the same element from executing.
    - `event.preventDefault()`: Cancels the browser's default behavior (e.g., following 
      a link or submitting a form) but does not stop propagation.

* Event Delegation:
  A pattern of attaching a single event listener to a parent element instead of adding 
  individual listeners to all children. Uses event bubbling to catch events from children.
  - Great for performance (saves memory) and automatically works for dynamically added 
    child elements.
  ```javascript
  document.getElementById("parent-list").addEventListener("click", function(event) {
    if (event.target && event.target.nodeName === "LI") {
      console.log("List item clicked:", event.target.textContent);
    }
  });
  ```


9. DATA STRUCTURES & COLLECTIONS [Importance: 8/10]
--------------------------------------------------------------------------------
* Array Methods: Map, Filter, Reduce:
  - `map()`: Creates a new array by applying a function to each element.
    ```javascript
    let double = arr.map(x => x * 2);
    ```
  - `filter()`: Creates a new array containing elements that pass a conditional check.
    ```javascript
    let evens = arr.filter(x => x % 2 === 0);
    ```
  - `reduce()`: Reduces an array to a single value by applying a reducer function.
    ```javascript
    let sum = numbers.reduce((accumulator, item) => accumulator + item, 0);
    ```

* Object and Array Destructuring:
  Extracted assignments from arrays and objects:
  ```javascript
  // Object:
  const classDetails = { strength: 78, benches: 39 };
  const { strength: classStrength, benches: classBenches } = classDetails;
  
  // Array:
  const arr = [1, 2, 3];
  const [first, second] = arr;
  ```

* Rest Parameter (`...` in declarations) vs. Spread Operator (`...` in execution):
  - Rest Parameter: Gathers parameters into an array. Must be the last parameter.
    ```javascript
    function sumAll(...args) { return args.reduce((a, b) => a + b); }
    ```
  - Spread Operator: Spreads array elements or object properties.
    ```javascript
    let arr2 = [...arr1, 4, 5];
    ```

* WeakSet & WeakMap:
  - `WeakSet`: Stores unique objects. Objects are held weakly, meaning they can be 
    garbage-collected if there are no other references to them.
  - `WeakMap`: Stores key/value pairs where keys must be objects (or unique symbols) 
    and values can be any type. Keys are held weakly, preventing memory leaks if the key 
    objects are deleted elsewhere. Non-enumerable.


10. REGULAR EXPRESSIONS & STRING UTILITIES [Importance: 6/10]
--------------------------------------------------------------------------------
* charAt(index):
  Returns the character at the specified index in a string.

* RegExp Methods: `exec()` vs. `test()`:
  - `exec()`: Searches a string for a match and returns an array containing matched 
    groups, or `null` if no match is found.
    ```javascript
    let match = /x/.exec("xyz"); // Returns array with match details
    ```
  - `test()`: Tests for a match and returns boolean `true` or `false`.
    ```javascript
    let exists = /x/.test("xyz"); // Returns true
    ```


11. ADVANCED OPTIMIZATION TECHNIQUES [Importance: 9/10]
--------------------------------------------------------------------------------
* Memoization:
  An optimization technique that caches function computation results.
  ```javascript
  // Fibonacci Memoization Example:
  function memoizedFib() {
    const cache = {};
    return function fib(n) {
      if (n in cache) return cache[n];
      if (n <= 1) return n;
      cache[n] = fib(n - 1) + fib(n - 2);
      return cache[n];
    };
  }
  const fib = memoizedFib();
  console.log(fib(40)); // Much faster than standard recursion
  ```

* Generators:
  Functions that can pause and resume execution using `function*` and `yield`.
  - Pause execution and return value on the fly without storing all results in memory.
  - Yields object format: `{ value: X, done: boolean }`.
  - Can safely support infinite loop generators without freezing execution.
  ```javascript
  function* idGenerator() {
    let id = 1;
    while (true) {
      yield id++;
    }
  }
  const gen = idGenerator();
  console.log(gen.next().value); // 1
  console.log(gen.next().value); // 2
  console.log(gen.next().value); // 3
  ```
  - Comparison: Async/Await vs. Generators:
    - Generators execute yield-by-yield manually when `.next()` is called, yielding `{ value: X, done: boolean }` objects.
    - Async/Await functions execute sequentially to completion automatically, wrapping the final returned value in a Promise (or throwing an error).
    - Async/Await is a specialized implementation designed specifically for cleaner sequential asynchronous control flow.

* Debouncing & Throttling:
  - Debouncing: Runs the function only after a certain delay has elapsed since the 
    last invocation (e.g. typing search queries).
    ```javascript
    function debounce(fn, delay) {
      let timerId = null;
      return function (...args) {
        if (timerId) clearTimeout(timerId);
        timerId = setTimeout(() => {
          fn.apply(this, args);
        }, delay);
      };
    }
    ```
  - Throttling: Limits execution to at most once every X milliseconds (e.g. scroll handlers).
    ```javascript
    function throttle(fn, delay) {
      let lastExecutionTime = 0;
      return function (...args) {
        const now = Date.now();
        if (now - lastExecutionTime >= delay) {
          lastExecutionTime = now;
          fn.apply(this, args);
        }
      };
    }
    ```


12. MAP vs. PLAIN OBJECT
--------------------------------------------------------------------------------
* `Map`:
  A keyed collection where any value (including objects) can be a key.
  - Maintains insertion order.
  - Has a built-in `.size` property.
  - Never has key collisions from prototype properties.
  - Iterable directly with `for..of`.
  ```javascript
  const map = new Map();
  map.set('name', 'Alice');
  map.set(42, 'answer');
  map.set({}, 'obj-key');
  console.log(map.size); // 3
  ```

* When to use `Map` over `Object`:
  - Keys are not strings/symbols (e.g., DOM nodes, objects).
  - Key insertion order must be guaranteed.
  - Frequent add/delete operations (Map is optimized for this).
  - Use plain `Object` when keys are strings and you need JSON serialization.


13. WEAKREF [Importance: 5/10]
--------------------------------------------------------------------------------
* `WeakRef`:
  Holds a weak reference to an object without preventing it from being garbage-collected.
  - Use `.deref()` to access the target; returns `undefined` if it has been GC'd.
  ```javascript
  let obj = { name: 'target' };
  const ref = new WeakRef(obj);
  obj = null; // Original reference dropped

  const target = ref.deref();
  if (target) {
    console.log(target.name); // 'target' (if not yet GC'd)
  }
  ```
  - Pairs naturally with `WeakMap`/`WeakSet` for cache/registry patterns.
  - Do not rely on `WeakRef` for critical program logic — GC timing is non-deterministic.


14. PROXY & REFLECT [Importance: 7/10]
--------------------------------------------------------------------------------
* Proxy:
  Wraps an object and intercepts fundamental operations (get, set, delete, etc.) via traps.
  ```javascript
  const handler = {
    get(target, prop) {
      return prop in target ? target[prop] : `Property '${prop}' not found`;
    },
    set(target, prop, value) {
      if (typeof value !== 'number') throw new TypeError('Only numbers allowed');
      target[prop] = value;
      return true;
    }
  };

  const proxy = new Proxy({}, handler);
  proxy.age = 25;
  console.log(proxy.age);    // 25
  console.log(proxy.name);   // "Property 'name' not found"
  ```

* Reflect:
  A built-in object providing methods matching the Proxy trap signatures. Used to 
  forward operations to the original target inside traps.
  ```javascript
  get(target, prop, receiver) {
    return Reflect.get(target, prop, receiver); // Correct forwarding
  }
  ```
  - Key insight: `Reflect` methods mirror Proxy traps 1-to-1, making them natural partners.


15. MODERN ARRAY UTILITIES [Importance: 8.5/10]
--------------------------------------------------------------------------------
* `Array.at(index)`:
  Accepts positive and negative integers. Negative integers count back from the end.
  ```javascript
  const arr = [1, 2, 3, 4, 5];
  console.log(arr.at(0));   // 1
  console.log(arr.at(-1));  // 5 (last element)
  console.log(arr.at(-2));  // 4
  ```
  Advantage over `arr[arr.length - 1]`: cleaner and works on strings/TypedArrays too.

* `Array.from()`:
  Creates an array from an iterable or array-like object.
  ```javascript
  Array.from('hello');           // ['h', 'e', 'l', 'l', 'o']
  Array.from({length: 3}, (_, i) => i); // [0, 1, 2]
  ```

* `Array.findIndex()` vs `Array.indexOf()`:
  - `indexOf(val)`: finds by strict equality.
  - `findIndex(fn)`: finds by predicate — works for objects and complex conditions.
  ```javascript
  const users = [{ id: 1 }, { id: 2 }];
  users.findIndex(u => u.id === 2); // 1
  ```


16. ES MODULES (ESM) [Importance: 8/10]
--------------------------------------------------------------------------------
* Named vs. Default Exports:
  ```javascript
  // Named exports — multiple per file:
  export const PI = 3.14;
  export function add(a, b) { return a + b; }

  // Default export — one per file:
  export default class MyClass { ... }
  ```

* Importing:
  ```javascript
  import { PI, add } from './math.js';       // Named
  import MyClass from './MyClass.js';          // Default
  import * as math from './math.js';           // Namespace import
  import { add as sum } from './math.js';      // Aliased
  ```

* Dynamic Import (lazy loading):
  Returns a Promise — useful for code splitting.
  ```javascript
  const { add } = await import('./math.js');
  ```

* ESM vs. CommonJS (CJS):
  | Feature            | ESM (`import/export`)   | CJS (`require/module.exports`) |
  |--------------------|-------------------------|--------------------------------|
  | Environment        | Browser + Node.js       | Node.js (legacy)               |
  | Loading            | Static (compile-time)   | Dynamic (runtime)              |
  | `this` at top level| `undefined`             | `module.exports`               |
  | Tree-shakeable     | Yes                     | No                             |


17. ERROR HANDLING PATTERNS [Importance: 7.5/10]
--------------------------------------------------------------------------------
* Runtime Error Types:
  - `ReferenceError`: Accessed a non-existent variable.
  - `TypeError`: Value is not the expected type.
  - `SyntaxError`: Code structure is invalid.
  - `RangeError`: Number is outside the allowed range.

* try / catch / finally:
  ```javascript
  try {
    riskyOperation();
  } catch (err) {
    console.error(err.message);
  } finally {
    cleanup(); // Always runs, even if an error was thrown
  }
  ```

* Custom Error Classes:
  Extend the built-in `Error` class to create domain-specific error types.
  ```javascript
  class ValidationError extends Error {
    constructor(message, field) {
      super(message);
      this.name = 'ValidationError';
      this.field = field;
    }
  }

  try {
    throw new ValidationError('Required field missing', 'email');
  } catch (err) {
    if (err instanceof ValidationError) {
      console.log(`${err.name} on '${err.field}': ${err.message}`);
    }
  }
  ```

* Async Error Handling:
  ```javascript
  // Option 1: try/catch with async/await
  async function fetchUser(id) {
    try {
      const res = await fetch(`/api/users/${id}`);
      if (!res.ok) throw new Error(`HTTP error: ${res.status}`);
      return await res.json();
    } catch (err) {
      console.error('Fetch failed:', err);
    }
  }

  // Option 2: .catch() on the promise chain
  fetch('/api/data')
    .then(res => res.json())
    .catch(err => console.error(err));
  ```

* Unhandled Promise Rejections:
  Always handle rejections. In Node.js/browsers, unhandled rejections can crash the 
  process or print warnings:
  ```javascript
  process.on('unhandledRejection', (reason) => {
    console.error('Unhandled rejection:', reason);
  });
  ```
