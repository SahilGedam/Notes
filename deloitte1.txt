# JavaScript Interview Questions & Answers (3 Years Angular Developer)

# 1. How many ways can we create objects in JavaScript?

There are multiple ways to create objects in JavaScript:

1. Object Literal (Most Common)
```javascript
const person = {
  name: "John",
  age: 25
};
```

2. Object Constructor
```javascript
const person = new Object();
person.name = "John";
```

3. Constructor Function
```javascript
function Person(name) {
    this.name = name;
}

const p1 = new Person("John");
```

4. ES6 Class
```javascript
class Person {
    constructor(name) {
        this.name = name;
    }
}

const p = new Person("John");
```

5. Object.create()
```javascript
const person = {
    greet() {
        console.log("Hello");
    }
};

const obj = Object.create(person);
```

6. Singleton Object
```javascript
const Config = {
    apiUrl: "localhost"
};
```

Interview Tip:
Object Literal is the most commonly used way in real projects.

# 2. What is IIFE and why is it used?

IIFE stands for Immediately Invoked Function Expression.

It executes immediately after it is defined.

Example:

```javascript
(function () {
    console.log("Executed");
})();
```

Why do we use it?

- Avoid polluting global scope
- Create private variables
- Execute initialization code immediately
- Before ES6 modules, it was commonly used for encapsulation

# 3. Explain Closure

Closure means an inner function remembers variables of its outer function even after the outer function has finished execution.

Example:

```javascript
function counter() {
    let count = 0;

    return function () {
        count++;
        return count;
    };
}

const increment = counter();

console.log(increment()); //1
console.log(increment()); //2
```

Uses:

- Data hiding
- Private variables
- Event handlers
- Callbacks
- Memoization

# 4. Explain JavaScript Execution

JavaScript executes code in two phases.

1. Creation Phase
- Memory allocation happens
- Variables are initialized
- Functions are stored completely in memory
2. Execution Phase
- Code executes line by line
- Variable values are assigned
- Functions are called

JavaScript Runtime consists of:

- Call Stack
- Heap Memory
- Web APIs
- Callback Queue
- Microtask Queue
- Event Loop

The Event Loop continuously checks whether the Call Stack is empty. If it is empty, it moves tasks from the queues into the Call Stack.

# 5. Difference between let and var

var

- Function scoped
- Can redeclare
- Hoisted and initialized as undefined
- Can access before declaration
- Introduced in ES5

let

- Block scoped
- Cannot redeclare in same scope
- Hoisted but remains in Temporal Dead Zone
- Cannot access before declaration
- Introduced in ES6

Example:

```javascript
if (true) {
    var a = 10;
    let b = 20;
}

console.log(a); //10
console.log(b); //ReferenceError
```

# 6. Explain First Class Functions

JavaScript treats functions as first-class citizens.

Functions can:

- Be stored in variables
- Passed as arguments
- Returned from another function
- Stored inside objects

Example:

```javascript
function greet() {
    console.log("Hello");
}

function execute(fn) {
    fn();
}

execute(greet);
```

# 7. Difference between slice() and splice()

slice()

- Returns new array
- Original array remains unchanged

Example:

```javascript
const arr = [1,2,3,4];

const result = arr.slice(1,3);

console.log(result);
// [2,3]
```

splice()

- Modifies original array
- Can add or remove elements

Example:

```javascript
const arr = [1,2,3,4];

arr.splice(1,2);

console.log(arr);
// [1,4]
```

# 8. What is Prototype Chain?

Every JavaScript object has a hidden reference to another object called its prototype.

If a property is not found in the current object, JavaScript searches in its prototype.

Example:

```javascript
const animal = {
    eats: true
};

const dog = Object.create(animal);

console.log(dog.eats);
```

Prototype lookup:

dog
↓
animal
↓
Object.prototype
↓
null
# 9. What is Function Currying?

Currying converts a function with multiple parameters into multiple nested functions with one parameter each.

Example:

```javascript
function add(a) {
    return function (b) {
        return a + b;
    };
}

console.log(add(2)(5));
```

Advantages:

- Reusability
- Partial Application
- Cleaner Functional Programming

# 10. Difference between call(), apply() and bind()

All three methods are used to change the value of "this".

call()

Arguments passed individually.

```javascript
person.call(obj,1,2);
```

apply()

Arguments passed as array.

```javascript
person.apply(obj,[1,2]);
```

bind()

Returns a new function.

```javascript
const fn = person.bind(obj);

fn();
```

# 15. Explain Interceptors

Interceptors intercept every outgoing HTTP request and incoming HTTP response.

Uses:

- Add JWT Token
- Logging
- Error Handling
- Refresh Token
- Request Modification

Example:

```typescript
intercept(req, next) {

    const cloned = req.clone({
        setHeaders: {
            Authorization: `Bearer ${token}`
        }
    });

    return next.handle(cloned);
}
```

# 17. Error Handling in Observables

Use catchError operator.

Example:

```typescript
this.userService.getUsers().pipe(

    catchError(error => {

        console.error(error);

        return throwError(() => error);

    })

).subscribe();
```

For global API errors, we generally use HTTP Interceptors.

# 32. What is NgRx Store?

NgRx is a state management library based on the Redux pattern.

Main building blocks:

- Store → Holds application state
- Actions → Describe events
- Reducers → Update state
- Selectors → Read state
- Effects → Handle API calls and other side effects

Use NgRx when multiple components need to share complex state or when state management becomes difficult with services alone.
